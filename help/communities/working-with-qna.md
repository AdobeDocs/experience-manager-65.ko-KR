# Q&amp;A 포럼 기능{#q-a-forum-feature}

## 소개 {#introduction}

QnA(질문과 대답) 포럼 기능은 커뮤니티 구성원이 질문을 하고 답할 수 있는 영역을 제공합니다. 이를 통해 구성원은 다음과 같은 작업을 수행할 수 있습니다.

* 질문 만들기
* 인라인 이미지 추가(드래그 앤 드롭 지원)
* 질문 보기 및 답변
* 질문 검색
* QnA 콘텐츠 중재 도움말
* 모범 응답 식별
* 한 페이지에서 다른 페이지로 QnA 질문 이동

이 설명서에서는 다음 사항을 설명합니다.

* AEM 사이트에 QnA 포럼 기능 추가.
* `QnA`구성 요소에 대한 구성 설정입니다.

## 페이지에 Q&amp;A 포럼 추가 {#adding-a-q-a-forum-to-a-page}

작성자 모드에서 페이지에 `QnA` 구성 요소를 추가하려면 구성 요소 브라우저를 사용하여 `Communities / QnA`을(를) 찾아 QnA 포럼이 표시되어야 하는 페이지에 있는 위치로 드래그합니다.

필요한 정보는 [커뮤니티 구성 요소 기본 사항](/help/communities/basics.md)을 참조하세요.

[필수 클라이언트측 라이브러리](/help/communities/qna-essentials.md#essentials-for-client-side)가 포함된 경우 `QnA` 구성 요소는 다음과 같이 표시됩니다.

![qna-component](assets/qna-component.png)

### QnA 구성 {#configuring-qna}

배치된 `QnA` 구성 요소를 선택하여 편집 대화 상자를 여는 `Configure` 아이콘에 액세스하고 선택합니다.

![구성](assets/configure-new.png)

![qna-config](assets/qna-config.png)

#### 설정 탭 {#settings-tab}

**설정** 탭에서 주제(질문) 및 답변(답변)에 대한 설정을 지정합니다.

* **첨부 파일 썸네일 허용**

  선택하면 첨부된 이미지의 썸네일이 만들어집니다.

* **최대 첨부 썸네일 크기**

  첨부 파일 썸네일 이미지의 최대 크기(픽셀 단위)입니다. 기본값은 800 x 800입니다.

* 썸네일에 대한 **최소 이미지 크기**

  인라인 이미지에 대한 썸네일을 생성하기 위한 이미지의 최소 크기(바이트)입니다. 기본값은 100000바이트(100kb)입니다.

* **최대 썸네일 크기**

  인라인 이미지에 대한 썸네일 이미지의 최대 크기(픽셀 단위)입니다. 기본값은 800 x 800입니다.

* **페이지당 항목 수**

  페이지 하나에 표시되는 질문/게시물 수를 정의합니다. 기본값은 10입니다.

* **중재됨**

  선택하는 경우, 게시물 사이트에 게시하기 전에 주제 및 댓글 게시를 승인해야 합니다. 기본값은 선택 해제되어 있습니다.

* **닫힘**

  선택하면 포럼에 새로운 질문과 의견이 표시되지 않습니다. 기본값은 선택 해제되어 있습니다.

* **리치 텍스트 편집기**

  선택하면 주제와 주석을 마크업과 함께 입력할 수 있습니다. 기본값은 선택 해제되어 있습니다.

* **태그 지정 허용**

  선택하면 구성원이 자신의 게시물에 태그 레이블을 추가할 수 있습니다(**태그 필드** 탭 참조). 기본값은 선택 해제되어 있습니다.

* **파일 업로드 허용**

  선택하면 질문을 첨부하거나 댓글에 첨부 파일을 추가할 수 있습니다. 기본값은 선택 해제되어 있습니다.

* **다음 허용**

  선택하면 포럼 게시물에 다음 기능을 포함시켜 구성원이 새 게시물의 [알림을](/help/communities/notifications.md)받을 수 있습니다. 기본값은 선택 해제되어 있습니다.

* **고정 허용**

  선택하면 포럼 주제를 주제 목록의 맨 위에 고정할 수 있습니다. 기본값은 선택 해제되어 있습니다.

* **이메일 구독 허용**

  선택하면 회원에게 전자 메일로 새 게시물을 알릴 수 있습니다([구독](/help/communities/subscriptions.md)). 허용 다음을 확인하고 [전자 메일을 구성](/help/communities/email.md)해야 합니다. 기본값은 선택 해제되어 있습니다.

* **최대 파일 크기**

  `Allow File Uploads`을(를) 선택한 경우에만 관련이 있습니다. 이 필드는 업로드된 파일의 크기(바이트)를 제한합니다. 기본값은 104857600(10Mb)입니다.

* **허용되는 파일 형식**

  `Allow File Uploads`을(를) 선택한 경우에만 관련이 있습니다. &quot;점&quot; 구분 기호가 있는 쉼표로 구분된 파일 확장자 목록입니다. 예: .jpg, .jpeg, .png, .doc, .docx, .pdf 지정된 파일 유형이 있으면 지정되지 않은 파일 유형을 업로드할 수 없습니다. **모든** 파일 형식이 허용되는 기본값이 지정되지 않았습니다.

* **이미지 첨부 파일 최대 크기**

  [파일 업로드 허용]이 선택된 경우에만 해당됩니다. 업로드된 이미지 파일이 가질 수 있는 최대 바이트 수입니다. 기본값은 2097152(2Mb)입니다.

* **답글 허용**

  선택하면 질문에 게시된 댓글에 답글을 달 수 있습니다. 기본값은 선택 해제되어 있습니다.

* **투표 허용**

  선택하는 경우 질문과 함께 투표 기능을 포함합니다. 기본값은 선택 해제되어 있습니다.

* **사용자가 댓글과 주제를 삭제하도록 허용**

  선택하면 구성원이 게시한 댓글과 질문을 삭제할 수 있습니다. 기본값은 선택 해제되어 있습니다.

* **권한이 있는 구성원 허용**

  선택하면 권한이 있는 구성원만 컨텐츠를 만들 수 있습니다.

* **작성자 편집 모드에서 사용자 생성 콘텐츠 차단**

  활성화되면 작성자 모드에서 편집하는 동안 사용자 생성 컨텐츠를 차단합니다.

* **선택한 대답을 맨 위로 이동**

  선택하면 표시된 첫 번째 답변이 선택한 답변입니다. 기본값은 선택 해제되어 있습니다.
* **배지 표시**

  선택하면 회원들의 블로그 항목과 함께 획득 및 할당된 [배지](/help/communities/implementing-scoring.md)를 표시합니다. 기본값은 선택 해제되어 있습니다.

* **추천 콘텐츠 허용**

  선택하면 해당 아이디어는 [추천 콘텐츠](/help/communities/featured.md)(으)로 식별됩니다. 기본값은 선택 해제되어 있습니다.

* **언급 사용**

  활성화된 경우, 등록된 커뮤니티 사용자가 다른 등록된 구성원(이름, 성, 사용자 이름 사용)을 식별하고 일반 @user-name 구문을 사용하여 해당 구성원에 태그를 지정할 수 있도록 합니다. 태그가 지정된 사용자는 해당 언급 내용에 대한 알림을 받습니다.

* **최대 언급**

  게시물에 허용된 최대 언급 수를 제한합니다. 기본값은 10입니다.

* **UI 언급 패턴**

  게시물에 등록된 사용자를 태그 지정(@mention)할 허용 패턴 문자열을 지정합니다. 예: `~{{familyName}}{{givenName}}`

#### 사용자 중재 탭 {#user-moderation-tab}

**사용자 중재** 탭에서 게시된 주제(질문)와 답변(사용자 생성 콘텐츠)을 관리하는 방법을 지정합니다. 자세한 내용은 [사용자 생성 콘텐츠 중재](/help/communities/moderate-ugc.md)를 참조하십시오.

* **답변 거부**

  선택하면 신뢰할 수 있는 구성원 중재자가 게시된 답변을 거부하고 공개 Q&amp;A 포럼에 답변이 표시되지 않도록 할 수 있습니다. 기본값은 선택 해제되어 있습니다.

* **주제 닫기/다시 열기**

  선택하면 신뢰할 수 있는 구성원 중재자가 질문(주제)을 닫아 추가 편집 및 답변을 하고 질문을 다시 열 수 있습니다. 기본값은 선택 해제되어 있습니다.

* **항목 이동**
선택하면 게시측 중재자의 질문 이동을 허용합니다. 기본값은 선택 해제되어 있습니다.

* **게시물 플래그 지정**

  선택하면 구성원이 다른 사람의 질문이나 답변을 부적절한 것으로 표시하도록 허용합니다. 기본값은 선택 해제되어 있습니다.

* **이유 목록 플래그 지정**

  선택하면 멤버가 드롭다운 목록에서 질문이나 답변을 부적절하다고 플래그하는 이유를 선택할 수 있습니다. 기본값은 선택 해제되어 있습니다.

* **사용자 지정 플래그 이유**

  선택하면 구성원이 질문을 게시하거나 답변이 부적절한 이유를 직접 입력할 수 있습니다. 기본값은 선택 해제되어 있습니다.

* **중재 임계값**

  중재자에게 통지하기 전에 구성원에게 질문이나 답변에 플래그를 지정해야 하는 횟수를 입력합니다. 기본값은 1(1회)입니다.

* **플래그 지정 제한**

  질문 또는 답변이 공개 보기에서 숨겨지기 전에 플래그를 지정해야 하는 횟수를 입력합니다. -1로 설정하면 플래그가 지정된 질문이 공개 보기에서 숨겨지지 않습니다. 그렇지 않으면 이 숫자는 중재 임계값보다 크거나 같아야 합니다. 기본값은 5입니다.

#### 태그 필드 탭 {#tag-field-tab}

**태그 필드** 탭에서 **설정** 탭에서 허용되는 경우 적용할 수 있는 태그는 선택한 네임스페이스에 따라 제한됩니다.

* **허용되는 네임스페이스**

  **설정** 탭에서 `Allow Tagging`을(를) 선택한 경우 관련성이 있습니다. 적용할 수 있는 태그는 선택한 네임스페이스 범주 내의 태그로 제한됩니다. 네임스페이스 목록에는 &quot;표준 태그&quot;(기본 네임스페이스)와 &quot;모든 태그 포함&quot;이 포함됩니다. 기본값은 선택 안 함으로, 모든 네임스페이스가 허용됩니다.

* **제안 한도**

  포럼에 게시하는 구성원에 대한 제안으로 표시할 태그 수를 입력합니다. **-**1 값은 제한이 없음을 의미합니다. 기본값은 0입니다.

#### 정렬 설정 탭 {#sort-settings-tab}

**정렬 설정** 탭에서 게시한 댓글이 표시될 때 정렬되는 방법을 지정합니다.

* **정렬 기준**

  허용되는 모든 정렬 선택 항목을 확인하십시오. `Newest, Oldest, Last Updated, Most Viewed, Most Active, Most Followed and Most Liked`. 기본값은 `Newest, Oldest, Last Updated`입니다.

* **기본값으로 설정**

  아래로 끌어다 놓아 선택한 정렬 옵션 중 하나를 선택하여 기본값으로 표시합니다. 기본값은 `Newest`입니다.

* **분석 정렬에 대한 시간 옵션 선택**

  드롭다운을 사용하여 `All, Last 24 Hours, Last 7 Days, Last 30 Days` 중 하나를 선택하십시오. 기본값은 `All`입니다.

## 사이트 방문자 경험 {#site-visitor-experience}

### 답변 확인 {#identifying-answers}

`Select Answer` 단추를 사용하여 하나의 대답을 정답 또는 유용한 답변으로 표시할 수 있습니다. 질문이 답변됨으로 표시되면 `Unmark Chosen Answer` 단추를 사용하여 첫 번째 질문을 선택 해제할 때까지 다른 답변을 선택할 수 없습니다.

실행 가능한 답변으로 선택하면 `Unmark Chosen Answer` 버튼을 사용하여 선택을 취소할 수 있습니다.

정답이 실행 가능한 답변으로 선택되면, 질문이 `Answered`되었다는 표시가 기본 QnA 페이지에서 질문 주제 옆에 표시됩니다.

#### 중재자 및 관리자 {#moderators-and-administrators}

로그인한 사용자에게 중재자 또는 관리자 권한이 있는 경우 질문 또는 답변을 작성한 사람에 관계없이 구성 요소의 구성에서 허용하는 중재 작업을 수행할 수 있습니다.

또한 답변을 식별할 수 있습니다.

#### 구성원 {#members}

사이트 방문자는 구성에 따라 다음과 같은 작업을 수행할 수 있습니다.

* Post에게 새로운 질문이 있습니다.
* 작성한 질문을 편집하거나 삭제합니다.
* 다른 구성원의 질문 또는 답변에 플래그를 지정합니다.
* 작성한 질문에 대한 답변을 식별합니다.

#### 익명 {#anonymous}

로그인하지 않은 사이트 방문자는 게시된 질문과 답변만 읽고, 지원되는 경우 번역할 수 있지만 질문 및 답변을 추가하거나 다른 사람의 게시물에 플래그를 지정할 수 없습니다.

## 추가 정보 {#additional-information}

자세한 내용은 개발자를 위한 [QnA Essentials](/help/communities/qna-essentials.md) 페이지에서 확인할 수 있습니다.

게시한 주제와 댓글의 중재에 대해서는 [사용자 생성 콘텐츠 중재](/help/communities/moderate-ugc.md)를 참조하십시오.

게시한 주제 및 댓글에 태그를 지정하려면 [사용자 생성 콘텐츠에 태그 지정](/help/communities/tag-ugc.md)을 참조하세요.
