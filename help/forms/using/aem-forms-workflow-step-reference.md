# OSGi의 Forms 중심 워크플로우 - 단계 참조{#forms-centric-workflow-on-osgi-step-reference}

## Forms Workflow 단계 {#forms-workflow-steps}

Forms 워크플로우 단계는 AEM 워크플로우에서 AEM Forms 관련 작업을 수행합니다. 이러한 단계를 통해 OSGi 기반의 Forms 중심 워크플로우를 기반으로 적응형 양식을 신속하게 작성할 수 있습니다. 이러한 워크플로우는 기본 검토 및 승인 워크플로우, 내부 및 방화벽 내 업무 프로세스를 개발하는 데 사용할 수 있습니다. Forms Workflow 단계를 사용하여 문서 서비스를 시작하고 Adobe Sign 서명 워크플로우와 통합하며 다른 AEM Forms 작업을 수행할 수도 있습니다. 워크플로우에서 이러한 단계를 사용하려면 [AEM Forms Add-on](https://www.adobe.com/go/learn_aemforms_documentation_63)이 필요합니다.

## 작업 단계 {#assign-task-step} 할당

할당 작업 단계에서는 작업을 생성하여 사용자 또는 그룹에 할당합니다. 구성 요소는 작업을 할당하는 동시에 작업에 대한 적응형 양식 또는 비대화형 PDF를 지정합니다. 적응형 양식은 사용자의 입력을 받아들이는 데 필요하며 비대화형 PDF 또는 읽기 전용 적응형 양식이 검토 전용 워크플로우에 사용됩니다.

구성 요소를 사용하여 작업의 동작을 제어할 수도 있습니다. 예를 들어 자동 기록 문서를 만들고, 특정 사용자 또는 그룹에 작업을 할당하고, 제출된 데이터의 경로를 지정하고, 미리 채울 데이터의 경로를 지정하고, 기본 작업을 지정합니다. 작업 할당 단계에는 다음 속성이 있습니다.

* **제목:** 작업의 제목입니다. 제목이 AEM 받은 편지함에 표시됩니다.
* **설명:** 작업에 수행 중인 작업에 대한 설명입니다. 이 정보는 공유 개발 환경에서 작업할 때 다른 프로세스 개발자에게 유용합니다.

* **축소판 경로:** 작업 축소판의 경로입니다. 경로를 지정하지 않은 경우 적응형 양식 기본 축소판이 표시되고 문서 of Record의 경우 기본 아이콘이 표시됩니다.
* **워크플로우 단계:** 워크플로우는 여러 단계를 가질 수 있습니다. 이러한 단계는 AEM 받은 편지함에 표시됩니다. 모델의 속성에서 이러한 단계를 정의할 수 있습니다(사이드 킥과 페이지 > 페이지 속성 > 단계).
* **우선 순위:** 선택한 우선 순위가 AEM 받은 편지함에 표시됩니다. 사용 가능한 옵션은 높음, 보통, 낮음입니다. 기본값은 중간입니다.
* **기한:** 작업이 지연된 것으로 표시된 일 또는 시간 수를 지정합니다. **오프**&#x200B;를 선택하면 작업이 지연으로 표시되지 않습니다. 작업 기한이 지난 후 특정 작업을 수행하도록 시간 초과 핸들러를 지정할 수도 있습니다.

* **일:** 작업을 완료하기 전 일 수입니다. 작업이 사용자에게 할당된 후 일 수가 계산됩니다. 작업이 완료되지 않고 일 필드에 지정한 일 수를 교차하는 경우, 이 옵션을 선택하면 기한 후 시간 초과 처리기가 트리거됩니다.
* **시간:** 작업이 완료되기 전 시간. 작업이 사용자에게 할당된 후 시간 수가 계산됩니다. 작업이 완료되지 않고 시간 필드에 지정한 시간 수를 교차하는 경우, 이 옵션을 선택하면 시간 초과 처리기가 기한 후 트리거됩니다.
* **기한 후 시간 초과: 시간 초과 처리기 선택 필드를** 활성화하려면 이 옵션을 선택합니다.
* **시간 초과 핸들러:** 할당 작업 단계가 기한 초과일 때 실행할 스크립트를 선택합니다. [apps]/fd/dashboard/scripts/timeoutHandler의 CRX 저장소에 배치된 스크립트를 선택할 수 있습니다. 지정된 경로가 crx-repository에 없습니다. 관리자가 해당 경로를 사용하기 전에 만듭니다.
* **작업 세부 사항의 마지막 작업에서 작업 및 주석 강조 표시:** 마지막으로 수행한 작업을 표시하고 작업의 작업 세부 사항 섹션에서 받은 주석을 표시하려면 이 옵션을 선택합니다.
* **유형:** 워크플로우를 시작할 때 채울 문서 유형을 선택합니다. 응용 양식, 읽기 전용 응용 양식, 비대화형 PDF 문서, 대화형 통신 에이전트 UI 또는 대화형 통신 웹 채널 문서를 선택할 수 있습니다.
* **적응형 양식 사용: 입력 적응형** 양식을 찾는 방법을 지정합니다. 유형 드롭다운 목록에서 적응형 양식 또는 읽기 전용 적응형 양식을 선택하는 경우 이 옵션을 사용할 수 있습니다. 워크플로우에 제출된 적응형 양식을 절대 경로에서 사용하거나 변수의 경로에서 사용할 수 있습니다. String 유형의 변수를 사용하여 경로를 지정할 수 있습니다.\
   여러 적응형 양식을 워크플로우에 연결할 수 있습니다. 따라서 사용 가능한 입력 방법을 사용하여 런타임에 적응형 양식을 지정할 수 있습니다.

* **대화형 통신 사용:** 입력 대화형 통신을 찾을 방법을 지정합니다. 워크플로우에 제출된 대화형 통신을 절대 경로에 사용하거나 변수의 경로에서 사용할 수 있습니다. String 유형의 변수를 사용하여 경로를 지정할 수 있습니다. 유형 드롭다운 목록에서 대화형 통신 에이전트 UI 또는 대화형 통신 웹 채널 문서를 선택하는 경우 이 옵션을 사용할 수 있습니다.

>[!NOTE]
>
>AEM 받은 편지함에서 Interactive Communications Agent UI에 액세스하려면 cm-agent-users 및 workflow-users 그룹 할당이 있어야 합니다.

* **적응형 양식 또는 대화형 커뮤니케이션 경로**:적응형 양식 또는 대화형 통신의 경로를 지정합니다. 절대 경로에서 사용할 수 있고, 작업 과정에 제출되는 적응형 양식이나 대화형 통신을 사용하거나, 문자열 데이터 유형의 변수에 저장된 경로에서 적응형 양식을 검색할 수 있습니다.
* **다음을 사용하여 입력 PDF 선택:** 비대화형 PDF 문서의 경로를 지정합니다. [유형] 필드에서 비대화형 PDF 문서를 선택하면 이 필드를 사용할 수 있습니다. 페이로드를 기준으로 하거나, 절대 경로에 저장하거나, 문서 데이터 유형의 변수를 사용하여 입력 PDF를 선택할 수 있습니다. 예: [Payload_Directory]/Workflow/PDF/credit-card.pdf. crx-repository에 경로가 없습니다. 관리자가 해당 경로를 사용하기 전에 만듭니다. [PDF 경로] 옵션을 사용하려면 [기록 문서] 옵션이 활성화되어 있거나 양식 템플릿 기반의 적응형 양식이 필요합니다.
* **완료된 작업의 경우 적응형 양식을 다음과 같이 렌더링합니다**.작업이 완료로 표시되면 적응형 양식을 읽기 전용 적응형 양식 또는 PDF 문서로 렌더링할 수 있습니다. 적응형 양식을 기록 문서로 렌더링하기 위해 기록 문서 옵션이 활성화되거나 양식 템플릿 기반 적응형 양식이 필요합니다.
* **사전 입력:** 아래 나열된 다음 필드는 작업에 입력으로 사용됩니다.

   * **다음을 사용하여 입력 데이터 파일 선택: 입력** 데이터 파일의 경로(.json,) xml, .doc 또는 양식 데이터 모델). 페이로드를 기준으로 하는 경로를 사용하여 입력 데이터 파일을 검색하거나 문서, XML 또는 JSON 데이터 유형의 변수에 저장된 파일을 검색할 수 있습니다. 예를 들어 이 파일에는 AEM 받은 편지함 애플리케이션을 통해 양식에 제출된 데이터가 들어 있습니다. 예제 경로는 [Payload_Directory]/workflow/data입니다.
   * **다음을 사용하여 입력 첨부 선택:** 해당 위치에서 사용할 수 있는 첨부 파일이 작업과 연관된 양식에 첨부됩니다. 경로는 항상 페이로드를 기준으로 합니다. 예제 경로는 [Payload_Directory]/attachments/
   * **입력 JSON 선택:** 페이로드를 기준으로 하거나 문서, JSON 또는 양식 데이터 모델 데이터 유형의 변수에 저장된 경로를 사용하여 입력 JSON 파일을 선택합니다. 유형 드롭다운 목록에서 대화형 통신 에이전트 UI 또는 대화형 통신 웹 채널 문서를 선택하는 경우 이 옵션을 사용할 수 있습니다.
   * **사용자 정의 프리플라이트 서비스 선택: 데이터를** 검색하고 대화형 통신 웹 채널 문서 또는 에이전트 UI를 프리플라이트 서비스를 선택합니다.

   * **위에서 선택한 대화형 통신 자동 완성 서비스 사용: [대화형 통신 사용] 드롭다운 목록에 정의된 대화형 통신 자동 완성 서비스를 사용하려면 이 옵션을** 사용합니다.
   * **요청 속성 매핑:** 요청 속성 매핑 섹션을 사용하여 요청 속성의  [이름과 값을 정의합니다](../../forms/using/work-with-form-data-model.md#bindargument). 요청에 지정된 속성 이름 및 값을 기반으로 데이터 소스에서 세부 사항을 검색합니다. 리터럴 값 또는 String 데이터 유형의 변수를 사용하여 요청 속성 값을 정의할 수 있습니다.\
      [유형] 드롭다운 목록에서 [대화형 통신 에이전트 UI] 또는 [대화형 통신 웹 채널 문서]를 선택한 경우에만 자동 완성 서비스 및 요청 속성 매핑 옵션을 사용할 수 있습니다.

* **제출된 정보:** 아래 나열된 다음 필드는 작업의 출력 위치로 사용됩니다.

   * **다음을 사용하여 출력 데이터 파일 저장:** 데이터 파일(.json,)을 저장합니다. xml, .doc 또는 양식 데이터 모델). 데이터 파일에는 연결된 양식을 통해 제출된 정보가 들어 있습니다. 페이로드를 기준으로 하는 경로를 사용하여 출력 데이터 파일을 저장하거나 문서, XML 또는 JSON 데이터 유형의 변수에 저장할 수 있습니다. 예를 들어 데이터가 파일인 [Payload_Directory]/Workflow/data가 있습니다.
   * **다음을 사용하여 첨부 파일 저장:** 작업에 제공된 양식 첨부 파일을 저장합니다. 페이로드를 기준으로 하는 경로를 사용하여 첨부 파일을 저장하거나 문서 데이터 유형의 변수 배열에 저장할 수 있습니다.
   * **기록 문서 파일을 저장할** 수 있는 경로: 예: [Payload_Directory]/DocumentofRecord/credit-card.pdf. 페이로드를 기준으로 하는 경로를 사용하여 기록 문서를 저장하거나 문서 데이터 유형의 변수에 저장할 수 있습니다. **페이로드 기준** 옵션을 선택하면 경로 필드가 비어 있으면 기록 문서가 생성되지 않습니다. 이 옵션은 유형 드롭다운 목록에서 적응형 양식을 선택한 경우에만 사용할 수 있습니다.

   * **다음을 사용하여 웹 채널 데이터 저장: 페이로드를** 기준으로 하는 경로를 사용하여 웹 채널 데이터 파일을 저장하거나 문서, JSON 또는 양식 데이터 모델 데이터 유형의 변수에 저장합니다. 이 옵션은 유형 드롭다운 목록에서 대화형 통신 에이전트 UI를 선택한 경우에만 사용할 수 있습니다.
   * **다음을 사용하여 PDF 문서 저장: 페이로드를** 기준으로 하는 경로를 사용하여 PDF 문서를 저장하거나 문서 데이터 유형의 변수에 저장할 수 있습니다. 이 옵션은 유형 드롭다운 목록에서 대화형 통신 에이전트 UI를 선택한 경우에만 사용할 수 있습니다.
   * **다음을 사용하여 레이아웃 템플릿 저장:** 페이로드를 기준으로 하는 경로를 사용하여 레이아웃 템플릿을 저장하거나 문서 데이터 유형의 변수에 저장합니다. [레이아웃 템플릿](../../forms/using/layout-design-details.md)은 Forms Designer를 사용하여 만드는 XDP 파일을 나타냅니다. 이 옵션은 유형 드롭다운 목록에서 대화형 통신 에이전트 UI를 선택한 경우에만 사용할 수 있습니다.

* **할당자 > 할당 옵션:** 사용자에게 작업을 할당할 방법을 지정합니다. [참가자 선택기] 스크립트를 사용하여 사용자 또는 그룹에 작업을 동적으로 할당하거나 특정 AEM 사용자 또는 그룹에 작업을 할당할 수 있습니다.
* **참가자 선택기:** 이 옵션은 [할당 옵션]  **필드에서 사용자 또는** 그룹에게 동적 옵션을 선택하면 사용할 수 있습니다. ECMAScript 또는 서비스를 사용하여 사용자 또는 그룹을 동적으로 선택할 수 있습니다. 자세한 내용은 [사용자에게 동적 워크플로 할당](https://helpx.adobe.com/experience-manager/kb/HowToAssignAWorkflowDynamicallyToParticipants.html) 및 [사용자 정의 Adobe Experience Manager 동적 참가자 단계 만들기](https://helpx.adobe.com/experience-manager/using/dynamic-steps.html)를 참조하십시오.

* **참가자:** 이 필드는 참가자 선택 필드에서  **[!UICONTROL com.adobe.granite.workflow.core.process.]** RandomParticipantChoseroption을 선택한  **경우에 사용할 수** 있습니다. 이 필드를 사용하면 RandomParticipantChooser 옵션에 대한 사용자 또는 그룹을 선택할 수 있습니다.

* **할당자:** 이 필드는 참가자 선택 필드에서  **[!UICONTROL com.adobe.fd.workspace.step.service.]** VariableParticipantChoseris를 선택한  **경우 사용할 수** 있습니다. 이 필드에서 문자열 데이터 유형의 변수를 선택하여 담당자를 정의할 수 있습니다.

* **인수:** 이 필드는 [참가자 선택기] 필드에서 RandomParticipantSelect 스크립트 이외의 스크립트를 선택한 경우에 사용할 수 있습니다. 이 필드를 사용하면 참가자 선택기 필드에서 선택한 스크립트에 대해 쉼표로 구분된 인수 목록을 제공할 수 있습니다.

* **사용자 또는 그룹:** 작업이 선택한 사용자 또는 그룹에 할당됩니다. 이 옵션은 **지정 옵션** 필드에서 **특정 사용자 또는 그룹 옵션**&#x200B;을 선택한 경우에 사용할 수 있습니다. 이 필드는 workflow-users 그룹의 모든 사용자 및 그룹을 나열합니다.\
   **사용자 또는 그룹** 드롭다운 메뉴에는 로그인한 사용자가 액세스할 수 있는 사용자 및 그룹이 나열됩니다. 사용자 이름 표시는 해당 특정 사용자에 대한 crx-repository의 **users** 노드에 액세스 권한이 있는지 여부에 따라 달라집니다.

* **할당자에게 이메일로 알림:** 할당자에게 이메일 알림을 보내려면 이 옵션을 선택합니다. 이러한 알림은 작업이 사용자에게 할당되면 전송됩니다. 이 옵션을 사용하기 전에 AEM 웹 콘솔에서 알림을 활성화합니다. 단계별 지침은 [할당 작업 단계에 대한 이메일 알림 구성을 참조하십시오](../../forms/using/aem-forms-workflow.md)

* **HTML 이메일 템플릿**:알림 이메일에 사용할 이메일 템플릿을 선택합니다. 템플릿을 편집하려면 crx-repository의 /libs/fd/dashboard/templates/email/htmlEmailTemplate.txt에 있는 파일을 수정합니다.
* **위임 허용:** AEM 받은 편지함은 로그인한 사용자에게 할당된 워크플로우를 다른 사용자에게 위임하는 옵션을 제공합니다. 동일한 그룹 내에서 또는 다른 그룹의 워크플로우 사용자에게 위임할 수 있습니다. 작업이 단일 사용자에게 할당되고 **할당자 그룹** 구성원에게 위임 허용 옵션이 선택된 경우 작업을 다른 사용자 또는 그룹에 위임할 수 없습니다.
* **공유 설정:** AEM 받은 편지함은 받은 편지함에서 단일 또는 모든 작업을 다른 사용자와 공유하는 옵션을 제공합니다.
   * **할당자가 받은 편지함에서 명시적으로 공유하도록 허용** 옵션을 선택한 경우 사용자는 작업을 클릭하여 다른 AEM 사용자와 공유할 수 있습니다.
   * **할당자가 받은 편지함 공유** 옵션을 통해 공유할 수 있도록 허용 옵션을 선택하고 사용자가 자신의 받은 편지함 항목을 공유하거나 다른 사용자가 자신의 받은 편지함 항목에 액세스할 수 있게 하는 경우, 앞서 언급한 옵션이 활성화된 작업만 다른 사용자와 공유됩니다.

* **작업 > 기본 작업:** 즉시 전송, 저장 및 재설정 작업을 사용할 수 있습니다. 기본적으로 모든 기본 작업이 활성화됩니다.
* **경로 변수:** 경로 변수의 이름입니다. route 변수는 AEM 받은 편지함에서 사용자가 선택하는 사용자 지정 작업을 캡처합니다.
* **경로:** 작업은 다른 경로로 분기할 수 있습니다. AEM Inbox에서 선택한 경우 경로는 값을 반환하고 선택한 경로를 기준으로 워크플로우가 분기됩니다. String 데이터 유형의 변수에 경로를 저장하거나 **Literal**&#x200B;을 선택하여 수동으로 경로를 추가할 수 있습니다.

* **제목**:경로의 제목을 지정합니다. AEM 받은 편지함에 표시됩니다.
* **Coral 아이콘**:산호 아이콘의 HTML 속성을 지정합니다. Adobe CorelUI 라이브러리는 터치 퍼스트 아이콘 세트를 제공합니다. 경로에 아이콘을 선택하고 사용할 수 있습니다. AEM 받은 편지함의 제목과 함께 표시됩니다. 경로에 경로를 변수에 저장할 경우 경로는 기본 &#39;태그&#39; 산호호 아이콘을 사용합니다.
* **할당자가 댓글을 추가할 수 있도록 허용**:작업에 대한 설명을 활성화하려면 이 옵션을 선택합니다. 할당자는 작업 제출 시 AEM 받은 편지함 내에서 주석을 추가할 수 있습니다.
* **변수에 주석 저장:** String 데이터 유형의 변수에 주석을 저장합니다. 이 옵션은 **할당자가 댓글 추가 허용** 확인란을 선택하는 경우에만 표시됩니다.

* **할당자가 작업에 첨부 파일을 추가할 수 있도록 허용**:작업에 대한 첨부 파일을 활성화하려면 이 옵션을 선택합니다. 할당자는 작업 제출 시 AEM 받은 편지함 내에서 첨부 파일을 추가할 수 있습니다.
* **다음을 사용하여 출력 작업 첨부 파일** 저장:첨부 폴더의 위치를 지정합니다. 페이로드 관련 경로 또는 문서 데이터 유형의 배열 변수를 사용하여 출력 작업 첨부 파일을 저장할 수 있습니다. 이 옵션은 **할당자가 작업** 확인란을 선택하고 **적응형 양식**, **읽기 전용 적응형 양식** 또는 **비대화형 PDF 문서******&#x200B;양식/문서&#x200B;**탭의 아래쪽 목록.**

>[!NOTE]
>
>런타임 시 에이전트 UI의 [첨부 파일] 탭을 사용하여 첨부 파일을 대화형 통신에 연결합니다. 연결된 첨부 파일은 전체 상태에서 작업 항목을 연 후 사이드 킥에 작업 첨부 파일로 표시됩니다.

* **사용자 지정 메타데이터 사용:** 사용자 지정 메타데이터 필드를 활성화하려면 이 옵션을 선택합니다. 사용자 지정 메타데이터는 이메일 템플릿에 사용됩니다.
* **사용자 지정 메타데이터:** 이메일 템플릿의 사용자 지정 메타데이터를 선택합니다. 사용자 지정 메타데이터는 apps/fd/dashboard/scripts/metadataScripts의 crx-repository에서 사용할 수 있습니다. 지정된 경로가 crx-repository에 없습니다. 관리자가 해당 경로를 사용하기 전에 만듭니다. 사용자 지정 메타데이터에 서비스를 사용할 수도 있습니다. 사용자 지정 메타데이터를 제공하도록 WorkitemUserMetadataService 인터페이스를 확장할 수도 있습니다.
* **이전 단계의 데이터 표시**:가능한 경우 이 옵션을 선택하여 응답자가 이전 할당자, 작업에 이미 수행한 작업, 작업에 추가된 댓글, 완료된 작업의 기록 문서를 볼 수 있도록 합니다.
* **이후 단계의 데이터 표시:** 현재 할당자가 후속 할당자가 수행한 작업 및 작업에 추가된 주석을 볼 수 있도록 하려면 이 옵션을 선택합니다. 또한 현재 할당자는 가능한 경우 완료된 작업의 기록 문서를 볼 수 있습니다.
* **데이터 유형 가시성:** 기본적으로 할당자는 이전 및 이후 할당자가 추가한 기록 문서, 할당자, 수행한 작업 및 주석을 볼 수 있습니다. 데이터 유형 옵션의 가시성을 사용하여 할당자에게 표시되는 데이터 유형을 제한합니다.

## 이메일 보내기 단계 {#send-email-step}

이메일 단계를 사용하여 기록 문서, 적응형 양식의 링크, 인터랙티브한 통신 링크 또는 첨부된 PDF 문서와 함께 이메일을 보낼 수 있습니다. 이메일 보내기 단계는 [HTML 이메일](https://en.wikipedia.org/wiki/HTML_email)을 지원합니다. HTML 이메일은 반응형으로 수신자의 이메일 클라이언트 및 화면 크기에 맞게 조정됩니다. HTML 이메일 템플릿을 사용하여 이메일의 모양, 색상 구성표 및 비헤이비어를 정의할 수 있습니다.

이메일 단계에서는 Day CQ Mail Service를 사용하여 이메일을 보냅니다. 이메일 단계를 사용하기 전에 [이메일 서비스](../../forms/using/aem-forms-workflow.md)가 구성되어 있는지 확인하십시오. 이메일 단계에는 다음 속성이 있습니다.

**제목:** 단계의 제목은 워크플로우 편집기에서 단계를 식별하는 데 도움이 됩니다.

**설명:** 설명은 공유 개발 환경에서 작업할 때 다른 프로세스 개발자에게 유용합니다.

**이메일 제목:** 피사체는 워크플로우 메타데이터에서 검색하거나 수동으로 지정하거나 변수에 저장된 값에서 검색할 수 있습니다. 다음 옵션 중에서 선택합니다.

* **문자 -** 제목을 수동으로 지정합니다.
* **워크플로우 메타데이터에서 검색**  - 메타데이터 속성에서 제목을 검색합니다.
* **변수**  - 문자열 데이터 유형의 변수에 저장된 값에서 제목을 검색합니다.

**HTML 이메일 템플릿**:이메일의 HTML 템플릿입니다. 이메일 템플릿에서 변수를 지정할 수 있습니다. 이메일 단계는 입력에 대한 템플릿에 포함된 모든 변수를 추출하고 표시합니다.

**이메일 템플릿 메타데이터:** 이메일 템플릿 변수의 값은 사용자가 지정한 값, 작성자 또는 게시 서버의 자산 경로, 이미지 또는 워크플로우 메타데이터 속성일 수 있습니다.

* **리터럴:** 지정할 정확한 값을 알고 있을 때 옵션을 사용합니다. 예: [example@example.com](mailto:example@example.com).

* **워크플로우 메타데이터:** 사용할 값이 워크플로우 메타데이터 속성에 저장될 때 이 옵션을 사용합니다. 옵션을 선택한 후 [워크플로우 메타데이터] 옵션 아래의 빈 텍스트 상자에 메타데이터 속성 이름을 입력합니다. 예: emailAddress.
* **자산 URL:** 이 옵션을 사용하여 대화형 커뮤니케이션의 웹 링크를 이메일에 포함합니다. 옵션을 선택한 후 포함할 대화형 통신을 찾아보고 선택합니다. 자산은 작성자 또는 게시 서버에 상주할 수 있습니다.
* **이미지:** 이메일에 이미지를 포함하려면 옵션을 사용합니다. 옵션을 선택한 후 이미지를 찾아 선택합니다. 이미지 옵션은 이메일 템플릿에서 사용할 수 있는 이미지 태그(&lt;img src=&quot;*&quot;/>)에만 사용할 수 있습니다.

**보낸 사람/받는 사람의 이메일 주소:** 리터럴옵션을  **** 선택하여 이메일 주소를 수동으로 지정하거나 워크플로우 메타데이터 **에서** 검색 옵션을 선택하여 메타데이터 속성에서 이메일 주소를 검색합니다. 워크플로 메타데이터에서 검색&#x200B;**옵션에 대한 메타데이터 속성 배열 목록을 지정할 수도 있습니다.** 문자열 데이터 유형의 변수에 저장된 값에서 전자 메일 주소를 검색하려면 **변수** 옵션을 선택합니다.

**첨부 파일:** 지정된 위치에서 사용 가능한 자산은 이메일에 첨부됩니다. 자산의 경로는 페이로드 또는 절대 경로를 기준으로 할 수 있습니다. 예제 경로는 [Payload_Directory]/attachments/입니다.

**변수** 옵션을 선택하여 문서, XML 또는 JSON 데이터 유형의 변수에 저장된 첨부 파일을 검색합니다.

**파일 이름:** 이메일 첨부 파일의 이름입니다. 이메일 단계에서는 첨부 파일의 원본 파일 이름을 지정된 파일 이름으로 변경합니다. 이름은 워크플로 메타데이터 속성 또는 변수에서 수동으로 지정하거나 검색할 수 있습니다. 지정할 정확한 값을 알고 있으면 **리터럴** 옵션을 사용합니다. **변수** 옵션을 사용하여 문자열 데이터 유형의 변수에 저장된 값에서 파일 이름을 검색합니다. 사용할 값이 워크플로 메타데이터 속성에 저장되면 **워크플로우 메타데이터에서 검색** 옵션을 사용합니다.

## 기록 문서 생성 단계 {#generate-document-of-record-step}

양식을 채우거나 제출하면 양식의 레코드를 인쇄 또는 문서 형식으로 보관할 수 있습니다. 이를 기록 문서(DoR)라고 합니다. 기록 문서 생성 단계를 사용하여 적응형 양식의 읽기 전용 또는 대화형 PDF 버전을 만들 수 있습니다. PDF 버전에는 적응형 양식의 레이아웃과 함께 양식에 입력된 정보가 포함되어 있습니다.

기록 문서 단계에는 다음 속성이 있습니다.

**적응형 양식 사용**:입력 적응형 양식을 찾을 방법을 지정합니다. 워크플로우에 제출된 적응형 양식을 절대 경로에서 사용하거나 변수의 경로에서 사용할 수 있습니다. String 데이터 유형의 변수를 사용하여 **Select 변수에 경로를 지정하여** 필드를 해결할 수 있습니다.\
여러 적응형 양식을 워크플로우에 연결할 수 있습니다. 따라서 사용 가능한 입력 방법을 사용하여 런타임에 적응형 양식을 지정할 수 있습니다.

**적응형 양식 경로**:적응형 양식의 경로를 지정합니다. 이 필드는 **적응형 양식 사용** 필드에서 **절대 경로** 옵션에서 사용할 수 있습니다.

**다음을 사용하여 입력 데이터 입력:** 적응형 양식의 입력 데이터의 경로를 선택합니다. 데이터를 페이로드를 기준으로 한 위치에 유지하거나, 데이터의 절대 경로를 지정하거나, 문서, JSON 또는 XML 데이터 유형의 변수에 저장된 데이터를 검색할 수 있습니다. 입력 데이터가 적응형 양식과 병합되어 기록 문서를 만듭니다.

**첨부 파일의 경로를 사용하여 첨부 파일** 입력 경로를 선택합니다. 이러한 첨부 파일은 기록 문서에 포함되어 있습니다. 첨부 파일을 페이로드를 기준으로 한 위치에 보관하거나, 첨부 파일의 절대 경로를 지정하거나, 문서 데이터 유형의 배열 변수에 저장된 첨부 파일을 가져올 수 있습니다.

첨부 파일 등의 폴더 경로를 지정하면 폴더에서 직접 사용할 수 있는 모든 파일이 기록 문서에 첨부됩니다. 지정된 첨부 경로에서 직접 사용할 수 있는 폴더에서 사용할 수 있는 파일이 있으면 해당 파일은 첨부 파일로 기록 문서에 포함됩니다. 직접 사용할 수 있는 폴더에 폴더가 있으면 해당 폴더를 건너뜁니다.

**아래 옵션을 사용하여 생성된 기록 문서 저장:** 기록 파일의 문서를 보관할 위치를 지정합니다. 페이로드 폴더를 덮어쓰거나, 기록 문서를 페이로드 디렉토리 내의 위치에 배치하거나, 문서 데이터 유형의 변수에 기록 문서를 저장하도록 선택할 수 있습니다.

**로케일**:기록 문서의 언어를 지정합니다. 드롭다운 목록에서 로케일을 선택하려면 **리터럴**&#x200B;을 선택하고 문자열 데이터 유형의 변수에 저장된 값에서 로케일을 검색하려면 **변수**&#x200B;를 선택합니다. 로케일 값을 변수에 저장하는 동안 로케일 코드를 정의해야 합니다. 예를 들어, 영어의 경우 **en_US**, 프랑스어의 경우 **fr_FR**&#x200B;을 지정합니다.

## 양식 데이터 모델 서비스 호출 단계 {#invoke-form-data-model-service-step}

[AEM Forms 데이터 통합](../../forms/using/data-integration.md)을 사용하여 서로 다른 데이터 소스를 구성하고 연결할 수 있습니다. 이러한 데이터 소스는 데이터베이스, 웹 서비스, REST 서비스, OData 서비스 및 CRM 솔루션일 수 있습니다. AEM Forms 데이터 통합을 사용하면 데이터 검색, 추가, 구성된 데이터베이스에 대한 업데이트 작업을 수행하는 다양한 서비스를 포함하는 양식 데이터 모델을 만들 수 있습니다. **데이터 모델 서비스 호출 단계**&#x200B;를 사용하여 양식 데이터 모델(FDM)을 선택하고 FDM의 서비스를 사용하여 서로 다른 데이터 소스에 데이터를 검색, 업데이트 또는 추가할 수 있습니다.

양식 데이터 모델 서비스 호출 단계에는 양식 데이터 모델 작업을 용이하게 하는 다음과 같은 필드가 있습니다.

* **제목:** 단계의 제목입니다. 워크플로우 편집기에서 단계를 식별하는 데 도움이 됩니다.
* **설명:** 공유 개발 환경에서 작업하는 경우 다른 프로세스 개발자에게 유용한 설명입니다.

* **양식 데이터 모델 경로**:서버에 있는 양식 데이터 모델을 찾아 선택합니다.

* **서비스**:선택한 양식 데이터 모델에서 제공하는 서비스 목록입니다.
* **서비스 입력 > 문자, 변수 또는 워크플로우 메타데이터 및 JSON 파일을 사용하여 입력 데이터 제공**:서비스에는 여러 개의 인수가 있을 수 있습니다. 워크플로 메타데이터 속성, JSON 개체, 변수 또는 제공된 텍스트 상자에 서비스 인수의 값을 직접 입력하거나 다음 옵션을 선택합니다.

   * **리터럴:** 지정할 정확한 값을 알고 있을 때 옵션을 사용합니다. 예: srose@we.info.
   * **변수:** 이 옵션을 사용하여 변수에 저장된 값을 검색합니다.
   * **워크플로우 메타데이터에서 검색:** 사용할 값이 워크플로우 메타데이터 속성에 저장될 때 이 옵션을 사용합니다. 예: emailAddress.
   * **JSON 점 표기법:** 사용할 값이 JSON 파일에 있을 때 이 옵션을 사용합니다. 예: insurance.customerDetails.emailAddress. JSON 점 표기법 옵션은 입력 JSON에서 입력 필드 매핑 옵션을 선택한 경우에만 사용할 수 있습니다.
   * **입력 JSON에서 입력 필드 매핑: JSON** 파일의 경로를 지정하여 JSON 파일에서 일부 서비스 인수의 입력 값을 얻습니다. JSON 파일의 경로는 페이로드, 절대 경로를 기준으로 하거나 JSON 또는 양식 데이터 모델 유형의 변수를 사용하여 입력 JSON 문서를 선택할 수 있습니다.

* **서비스 입력 > 변수 또는 JSON 파일을 사용하여 입력 데이터 제공:** 이 옵션을 선택하여 절대 경로에 저장된 JSON 파일, 페이로드 관련 경로 또는 변수에 저장된 모든 인수에 대한 값을 가져옵니다.
* **다음을 사용하여 JSON 문서 입력** 선택:모든 서비스 인수에 대한 값을 포함하는 JSON 파일입니다. JSON 파일의 경로는 페이로드&#x200B;**또는**&#x200B;절대 경로를 기준으로 **할 수 있습니다.** JSON 또는 양식 데이터 모델 데이터 유형의 변수를 사용하여 입력 JSON 문서를 검색할 수도 있습니다.

* **JSON 점 표기법:** 지정된 JSON 파일의 모든 객체를 서비스 인수에 대한 입력으로 사용하려면 필드를 비워 둡니다. 지정된 JSON 파일에서 서비스 인수에 대한 입력으로 특정 JSON 개체를 읽으려면 JSON 개체에 대한 점 표기법을 지정합니다. 예를 들어 섹션 시작 부분에 나열된 JSON과 유사한 JSON이 있는 경우 insurance.customerDetails를 지정하여 서비스에 대한 입력으로 고객의 모든 세부 정보를 제공합니다.
* **서비스 출력 > 출력 값을 변수 또는 메타데이터에 매핑 및 쓰기:** 출력 값을 crx-repository에 있는 워크플로우 인스턴스 메타데이터 노드의 속성으로 저장하는 옵션을 선택합니다. 메타데이터 속성의 이름을 지정하고 메타데이터 속성에 매핑할 해당 서비스 출력 속성을 선택합니다. 예를 들어 출력 서비스에서 반환한 phone_number 속성을 워크플로 메타데이터의 phone_number 속성에 매핑합니다. 마찬가지로 출력물을 긴 데이터 유형의 변수에 저장할 수 있습니다.매핑할 **[!UICONTROL 서비스 출력 속성에 대한 속성을 선택하면**[!UICONTROL &#x200B;출력을&#x200B;]**에 저장 옵션에 대해 선택한 속성의 데이터를 저장할 수 있는 변수만 채워집니다.]**

* **서비스 출력 > 출력을 변수 또는 JSON 파일에 저장:** 출력 값을 절대 경로, 페이로드를 기준으로 하는 경로 또는 변수에 저장하는 옵션을 선택합니다.
* **아래의 옵션을 사용하여 출력 JSON 문서 저장: 출력 JSON** 파일을 저장합니다. 출력 JSON 파일의 경로는 페이로드 또는 절대 경로를 기준으로 할 수 있습니다. JSON 또는 양식 데이터 모델 데이터 유형의 변수를 사용하여 출력 JSON 파일을 저장할 수도 있습니다.

## 문서 서명 단계 {#sign-document-step}

문서 서명 단계에서는 Adobe Sign을 사용하여 문서에 서명할 수 있습니다. 문서 서명 단계에는 다음 속성이 있습니다.

* **계약 이름:** 계약의 제목을 지정합니다. 계약 이름은 서명자에게 보내는 이메일의 제목 및 본문 텍스트의 일부가 됩니다. String 데이터 유형의 변수에 이름을 저장하거나 **Literal**&#x200B;을 선택하여 이름을 수동으로 추가할 수 있습니다.

* **로케일:** 이메일 및 확인 옵션에 언어를 지정합니다. String 데이터 유형의 변수에 로케일을 저장하거나 **Literal**&#x200B;을 선택하여 사용 가능한 옵션 목록에서 로케일을 선택할 수 있습니다. 로케일 값을 변수에 저장하는 동안 로케일 코드를 정의해야 합니다. 예를 들어, 영어의 경우 **en_US**, 프랑스어의 경우 **fr_FR**&#x200B;을 지정합니다.

* **Adobe Sign Cloud 구성**:Adobe Sign 클라우드 구성을 선택합니다. AEM Forms에 대해 Adobe Sign을 구성하지 않은 경우 [Adobe Sign을 AEM Forms](../../forms/using/adobe-sign-integration-adaptive-forms.md)과(와) 통합을 참조하십시오.

* **서명 시 사용할 문서 선택: 페이로드를** 기준으로 한 위치에서 문서를 선택하거나 페이로드를 문서로 사용하거나 문서의 절대 경로를 지정하거나 문서 데이터 유형의 변수에 저장된 문서를 검색할 수 있습니다.
* **마감일까지 일** 일 수: 마감일까지 일 필드에서 지정한 일 수에 대한 작업에 대한 활동이 없으면 문서가 기한( **경과된 기한)으로** 표시됩니다. 문서화된 내용이 서명을 위해 사용자에게 지정된 후 일 수가 계산됩니다.
* **미리 알림 이메일 빈도:** 매일 또는 매주 간격으로 미리 알림 이메일을 보낼 수 있습니다. 이 주는 문서화된 항목이 서명을 위해 사용자에게 할당된 날부터 카운트됩니다.
* **서명 프로세스:** 순차적 또는 임의의 순서로 문서에 서명할 수 있습니다. 서명을 위해 한 명의 서명자가 동시에 문서를 수신하게 되는 순서대로 차례로 문서에 서명합니다. 첫 번째 서명자가 문서 서명을 완료한 후 문서가 두 번째 서명자에게 전송되는 등의 작업을 수행할 수 있습니다. 동시에 여러 서명자가 한 번에 문서에 서명할 수 있습니다.
* **리디렉션 URL: 리디렉션 URL** 을 지정합니다. 문서가 서명되면 할당자를 URL로 리디렉션할 수 있습니다. 일반적으로 이 URL에는 감사 메시지 또는 추가 지침이 포함되어 있습니다.
* **워크플로우 단계:** 워크플로우는 여러 단계를 가질 수 있습니다. 이러한 단계는 AEM 받은 편지함에 표시됩니다. 모델의 속성에서 이러한 단계를 정의할 수 있습니다(사이드 킥과 페이지 > 페이지 속성 > 단계).
* **서명자 선택:** 문서의 서명자를 선택할 방법을 지정합니다. 사용자 또는 그룹에 워크플로우를 동적으로 할당하거나 서명자의 세부 사항을 수동으로 추가할 수 있습니다.
* **서명자를 선택할 수 있는 스크립트 또는 서비스:** 이 옵션은 서명자 선택 필드에서 동적 옵션이 선택되어 있는 경우에만 사용할 수 있습니다. ECMAScript 또는 서비스를 지정하여 문서의 서명자 및 확인 옵션을 선택할 수 있습니다.
* **서명자 세부 사항:** 이 옵션은 서명자 선택 필드에서 수동 옵션을 선택한 경우에만 사용할 수 있습니다. 이메일 주소를 지정하고 선택 사항 확인 메커니즘을 선택합니다. 2단계 확인 메커니즘을 선택하기 전에 구성된 Adobe Sign 계정에 대해 해당 확인 옵션이 활성화되어 있는지 확인하십시오. 문자열 데이터 유형의 변수를 사용하여 **[!UICONTROL 이메일]**, **[!UICONTROL 국가 코드]** 및 **[!UICONTROL 전화 번호]** 필드에 대한 값을 정의할 수 있습니다. **[!UICONTROL 2단계 확인]** 드롭다운 목록에서 **[!UICONTROL 전화 확인]**&#x200B;을 선택한 경우에만 **[!UICONTROL 국가 코드]** 및 **[!UICONTROL 전화 번호]** 필드가 표시됩니다.
* **상태 변수:** Adobe Sign 사용 문서는 문서의 서명 상태를 문자열 데이터 유형의 변수에 저장합니다. 상태 변수 이름(adobeSignStatus)을 지정합니다. 인스턴스의 상태 변수는 /etc/workflow/instances/&lt;server>/&lt;date-time>/&lt;workflow 모델 인스턴스>/workItems/&lt;node>/metaData에 있는 CRXDE에서 사용할 수 있습니다.
* **아래 옵션을 사용하여 서명된 문서 저장:** 서명된 문서를 유지할 위치를 지정합니다. 페이로드 파일을 덮어쓰거나, 서명된 문서를 페이로드 디렉토리 내의 위치에 배치하거나, 서명된 문서를 문서 유형의 변수에 저장하도록 선택할 수 있습니다.

## 문서 서비스 단계 {#document-services-steps}

AEM 다큐멘트 서비스는 PDF 문서를 작성, 취합 및 보안을 위한 일련의 서비스입니다. AEM Forms은 각 문서 서비스에 대해 별도의 AEM 워크플로우 단계를 제공합니다.

작업 할당, 전자 메일 보내기 및 문서 서명 등의 다른 AEM Forms 워크플로우 단계와 유사하게 모든 AEM 문서 서비스 단계에서 변수를 사용할 수 있습니다. 변수 만들기 및 관리에 대한 자세한 내용은 AEM 작업 과정의 [변수](../../forms/using/variable-in-aem-workflows.md)를 참조하십시오.

### 문서 타임스탬프 적용 단계 {#apply-document-time-stamp-step}

문서에 타임스탬프를 추가합니다. 입력 문서 경로, 입력 문서 이름, 내보낸 데이터를 저장할 위치 등 문서 세부 사항을 제공합니다. 기존 페이로드 파일을 덮어쓰거나, 다른 파일 이름을 선택하여 페이로드 폴더 아래의 다른 파일에 데이터를 저장하거나, 데이터에 대한 절대 경로를 제공하거나, 데이터를 문서 데이터 유형의 변수에 저장하도록 선택할 수 있습니다.

### 이미지 단계 {#convert-to-image-step}로 변환

PDF 문서를 이미지 목록으로 변환합니다. 지원되는 이미지 형식은 JPEG, JPEG2000, PNG 및 TIFF입니다. 다음 정보는 TIFF 이미지 변환에 적용됩니다.

* 여러 페이지의 TIFF 파일이 생성됩니다.
* 일부 주석은 TIFF 이미지에 포함되지 않습니다. Acrobat에서 모양을 생성하도록 요구하는 주석은 포함되지 않습니다.

### PDF/A 단계 {#convert-to-pdf-a-step}로 변환

제공된 옵션을 사용하여 PDF 문서를 PDF/A 형식으로 변환합니다. PDF/A 버전의 PDF(Portable Document Format)는 문서를 보관하고 장기간 보존하기 위해 전문적입니다.

### PS 단계 {#convert-to-ps-step}로 변환

PDF 문서를 PostScript로 변환 PostScript로 변환할 때 변환 작업을 사용하여 소스 문서와 PostScript 레벨 2 또는 3으로 변환할지 여부를 지정할 수 있습니다. PostScript 파일로 변환하는 PDF 문서는 비대화형이어야 합니다.

### 지정된 유형 단계 {#create-pdf-from-specified-type-step}에서 PDF 만들기

입력 파일에서 PDF 문서를 생성합니다. 입력 문서는 페이로드를 기준으로 하거나, 절대 경로를 가지고 있거나, 페이로드 자체이거나, 문서 데이터 유형의 변수에 저장할 수 있습니다.

### URL/HTML/ZIP 단계에서 PDF 만들기 {#create-pdf-from-url-html-zip-step}

제공된 URL, HTML 및 ZIP 파일에서 PDF 문서를 생성합니다.

### 데이터 내보내기 단계 {#export-data-step}

PDF forms 또는 XDP 파일의 데이터를 내보냅니다. 입력 문서 및 내보내기 데이터 형식의 파일 경로를 입력해야 합니다. 데이터 형식 내보내기 옵션은 자동, XDP 및 XmlData입니다.

### 지정된 유형 단계 {#export-pdf-to-specified-type-step} Export PDF

PDF 문서를 선택한 형식으로 변환합니다.

### 비대화형 PDF 생성 단계 {#generatenoninteractive}

비대화형 PDF를 생성합니다. 다양한 사용자 정의 옵션을 제공합니다.

>[!NOTE]
>
>변수를 사용하여 입력 문서에 대한 템플릿 파일을 지정할 수 있습니다. 템플릿 파일의 경로를 문자열 데이터 유형의 변수에 저장합니다.

### 데이터 가져오기 단계 {#import-data-step}

양식 데이터를 PDF 양식으로 취합합니다. 양식 데이터를 PDF 양식으로 가져올 수 있습니다.

### DCX 단계 {#invokeddx} 호출

지정된 입력 문서 맵에서 DCX 파일을 실행하고 조작된 PDF 문서를 반환합니다.

>[!NOTE]
>
>변수를 사용하여 입력 문서에 대한 DCX 파일을 지정할 수 있습니다. DCX 파일을 Document 또는 XML 데이터 유형의 변수에 저장합니다.

### Optimize PDF 단계 {#optimize-pdf-step}

크기를 줄여 PDF 파일을 최적화합니다. 이러한 변환의 결과는 원본 버전보다 작을 수 있는 PDF 파일입니다. 또한 이 작업은 최적화 매개 변수에 지정된 PDF 버전으로 PDF 문서를 변환합니다.

최적화 설정은 파일이 최적화된 방식을 지정합니다. 다음은 설정의 예입니다.

* Target PDF 버전
* JavaScript 작업 및 포함된 페이지 축소판과 같은 개체 삭제
* 주석 및 첨부 파일과 같은 사용자 데이터 삭제
* 유효하지 않거나 사용하지 않은 설정 삭제
* 압축되지 않은 데이터 압축 또는 보다 효율적인 압축 알고리즘 사용
* 포함된 글꼴 제거
* 투명도 값 설정

### PDF 양식 렌더링 단계 {#renderpdf}

XDP(Form Designer)에서 만든 양식을 PDF 양식으로 렌더링합니다.

>[!NOTE]
>
>변수를 사용하여 입력 문서에 대한 템플릿 파일을 지정할 수 있습니다. 템플릿 파일의 경로를 문자열 데이터 유형의 변수에 저장합니다.

### 보안 문서 단계 {#secure-document-step}

문서 암호화, 서명 및 인증 AEM Forms은 암호 기반 암호와 인증서 기반 암호화를 모두 지원합니다. 문서에 서명하는 다양한 알고리즘 중에서 선택할 수도 있습니다. 예: SHA-256 및 SH-512. 또한 워크플로우 단계를 사용하여 PDF 문서를 읽을 수 있습니다. 워크플로우 단계에서는 바코드 디코딩, 디지털 서명, PDF 데이터 가져오기 및 내보내기 및 기타 옵션을 활성화하는 옵션을 제공합니다.

### 프린터로 보내기 단계 {#send-to-printer-step}

문서를 프린터로 바로 전송할 수 있습니다. 다음과 같은 인쇄 액세스 메커니즘을 지원합니다.

* **직접 액세스 가능한 프린터**:동일한 컴퓨터에 설치되는 프린터를 직접 액세스 가능한 프린터라고 하며 컴퓨터의 이름은 프린터 호스트입니다. 이 유형의 프린터는 컴퓨터에 직접 연결된 로컬 프린터일 수 있습니다.
* **간접 액세스 가능한 프린터**:인쇄 서버에 설치된 프린터는 다른 컴퓨터에서 액세스합니다. UNIX® 인쇄 시스템(CUPS) 및 LPD(Line Printer Daemon) 프로토콜과 같은 기술을 사용하여 네트워크 프린터에 연결할 수 있습니다. 간접 액세스 가능한 프린터에 액세스하려면 인쇄 서버의 IP 또는 호스트 이름을 지정합니다. 네트워크에 LPD가 실행 중인 경우 이 메커니즘을 사용하여 문서를 LPD URI로 보낼 수 있습니다. 이 메커니즘을 사용하면 LPD가 실행 중인 네트워크에 연결된 모든 프린터로 문서를 라우팅할 수 있습니다.

### 인쇄된 출력 생성 단계 {#generatePrintedOutput}

이 단계에서는 양식 디자인 및 데이터 파일을 제공하는 PCL, PostScript, ZPL, IPL, TPCL 또는 DPL 출력을 생성합니다. 데이터 파일은 양식 디자인과 병합되며 인쇄 형식이 지정됩니다. 이 단계에서 생성된 출력을 프린터로 직접 전송하거나 파일로 저장할 수 있습니다. 응용 프로그램의 양식 디자인 또는 데이터를 사용하려면 이 단계를 사용하는 것이 좋습니다. 양식 디자인 또는 양식 디자인이 네트워크, 로컬 파일 시스템 또는 HTTP 위치에 있는 경우 generatePrintedOutput 작업 작업을 사용합니다.

예를 들어, 응용 프로그램에서는 양식 디자인을 데이터 파일과 병합해야 합니다. 이 데이터에는 수백 개의 레코드가 포함되어 있습니다. 또한 출력물이 ZPL을 지원하는 프린터로 전송되어야 합니다. 양식 디자인과 입력 데이터는 애플리케이션에 있습니다. generatePrintedOutput 작업을 사용하여 각 레코드를 양식 디자인과 병합하고 출력물을 ZPL을 지원하는 프린터로 보냅니다.

* **[!UICONTROL 다음을 사용하여 컨텐츠 루트 위치를 선택합니다]**.컨텐츠 루트는 양식 디자인에 사용되는 상대 자산을 검색할 저장소의 URI, 절대 참조 또는 위치를 지정하는 문자열 값입니다. 예를 들어 양식 디자인이 이미지를 상대적으로 참조하는 경우(예: ../myImage.gif) myImage.gif는 repository://에 있어야 합니다. 기본값은 저장소의 루트 수준을 가리키는 repository://입니다.

   응용 프로그램에서 에셋을 선택하는 경우 컨텐츠 루트 URI 경로에 올바른 구조가 있어야 합니다. 예를 들어 SampleApp이라는 응용 프로그램에서 양식을 선택하고 SampleApp/1.0/forms/Test.xdp에 배치하면 Content Root URI를 repository://administrator@password/Applications/SampleApp/1.0/forms/ 또는 repository:/Applications/SampleApp/1.0/forms/(권한이 null인 경우)로 지정해야 합니다. 컨텐츠 루트 URI를 이 방법으로 지정하면 양식에서 참조된 모든 자산의 경로가 이 URI에 대해 해결됩니다.

* **[!UICONTROL 다음을 사용하여 XCI 파일을 선택합니다]**.XCI 파일은 양식 디자인 요소에 사용되는 글꼴 및 기타 속성을 설명하는 데 사용됩니다. 페이로드를 기준으로 XCI 파일을 절대 경로로 유지하거나 문서 데이터 유형의 변수를 사용하여 보관할 수 있습니다.

* **[!UICONTROL 로케일]**:PDF 문서를 생성하는 데 사용되는 언어를 지정합니다. 리터럴 값을 제공하는 경우 목록에서 언어를 선택하거나 다음 값 중 하나를 선택합니다.
   * **서버 기본값을 사용하려면**:(기본값) AEM Forms 서버에 구성된 로케일 설정을 사용합니다. 로케일 설정은 관리 콘솔을 사용하여 구성됩니다. ([디자이너 도움말](http://www.adobe.com/go/learn_aemforms_designer_65)을 참조하십시오.)

   * **사용자 지정 값을 사용하려면**:리터럴 상자에 로케일 코드를 입력하거나 로케일 코드가 포함된 문자열 변수를 선택합니다. 지원되는 로케일 코드의 전체 목록은 http://java.sun.com/j2se/1.5.0/docs/guide/intl/locale.doc.html을 참조하십시오.

* **[!UICONTROL 사본]**:출력용으로 생성할 복사본 수를 지정하는 정수 값입니다. 기본값은 1입니다.

* **[!UICONTROL 양면 인쇄]**:양면 인쇄 또는 단면 인쇄 사용 여부를 지정하는 페이지 매김 값입니다. PostScript 및 PCL을 지원하는 프린터에서는 이 값을 사용합니다. 리터럴 값을 제공하는 경우 다음 값 중 하나를 선택합니다.
   * **[!UICONTROL 양면 긴 가장자리]**:긴 가장자리 페이지 매김을 사용하여 양면 인쇄 및 인쇄를 사용합니다.
   * **[!UICONTROL 양면 짧은 가장자리]**:짧은 가장자리 페이지 매김을 사용하여 양면 인쇄 및 인쇄를 사용할 수 있습니다.
   * **[!UICONTROL 단순]**:단면 인쇄 사용
