# 적응형 양식에 대한 기록 문서 생성{#generate-document-of-record-for-adaptive-forms}

## 개요 {#overview}

양식을 제출한 후 일반적으로 고객은 나중에 참조할 수 있도록 양식에 입력한 정보의 기록을 인쇄 또는 문서 형식으로 보관해야 합니다. 이를 기록 문서라고 합니다.

이 문서에서는 적응형 양식에 대한 기록 문서를 생성하는 방법에 대해 설명합니다.

>[!NOTE]
>
>XFA 기반 적응형 양식에서는 기록 문서 자동 생성이 지원되지 않습니다. 그러나 적응형 양식을 기록 문서로 만드는 데 사용되는 XDP를 사용할 수 있습니다.

## 적응형 양식 유형 및 기록 문서 {#adaptive-form-types-and-their-documents-of-record}

적응형 양식을 만들 때 양식 모델을 선택할 수 있습니다. 옵션은 다음과 같습니다.

* [양식](../../forms/using/creating-adaptive-form.md#create-an-adaptive-form-based-on-an-xfa-form-template)템플릿 적응형 양식의 XFA 템플릿을 선택할 수 있습니다. XFA 템플릿을 선택할 때 위에 설명된 대로 레코드 문서에 연결된 XDP 파일을 사용할 수 있습니다.

* [XML 스키마](../../forms/using/creating-adaptive-form.md#create-an-adaptive-form-based-on-xml-or-json-schema)적응형 양식의 XML 스키마 정의를 선택할 수 있습니다. 적응형 양식의 XML 스키마를 선택할 때 다음을 수행할 수 있습니다.

   * 기록 문서에 XFA 템플릿을 연결합니다. 관련 XFA 템플릿에서 적응형 양식과 동일한 XML 스키마를 사용하는지 확인
   * 기록 문서 자동 생성

* 없음양식 모델 없이 적응형 양식을 만들 수 있습니다. 적응형 양식에 대해 기록 문서가 자동으로 생성됩니다.

양식 모델을 선택할 때 레코드 템플릿 구성 문서 아래에서 사용할 수 있는 옵션을 사용하여 레코드 문서를 구성합니다. See [Document of Record Template Configuration](#document-of-record-template-configuration).

## 자동으로 생성된 기록 문서 {#automatically-generated-document-of-record}

기록 문서를 통해 고객은 제출된 양식을 인쇄용으로 보관할 수 있습니다. 기록 문서를 자동으로 생성할 때 양식을 변경할 때마다 해당 기록 문서가 즉시 업데이트됩니다. 예를 들어 미국 국가를 국가로 선택한 고객의 연령 필드를 제거합니다. 이러한 고객이 기록 문서를 생성할 때 기록 문서에 연령 필드가 표시되지 않습니다.

자동으로 생성된 기록 문서는 다음과 같은 이점이 있습니다.

* 데이터 바인딩이 필요합니다.
* 또한 제출 시 레코드 문서에서 제외로 표시된 필드를 자동으로 숨깁니다. 별도의 노력이 필요하지 않습니다.
* 기록 템플릿 문서를 디자인하는 데 걸리는 시간을 절약할 수 있습니다.
* 또한 서로 다른 기본 템플릿을 사용하여 서로 다른 스타일 및 모양을 시도해 보고 기록 문서에 가장 적합한 스타일과 모양을 선택할 수 있습니다. 스타일 모양은 선택 사항이며 스타일을 지정하지 않으면 시스템 스타일이 기본값으로 설정됩니다.
* 이렇게 하면 양식의 변경 사항이 기록 문서에 즉시 반영됩니다.

**적응형 양식기록** 문서를 생성할 적응형 양식입니다.

기록 [문서의 기본 템플릿을 참조하십시오.](#base-template-of-a-document-of-record)

**적응형 양식에서 생성된 레코드 템플릿** XFA 템플릿(XDP 파일) 문서

See [Document of Record Template Configuration](#document-of-record-template-configuration).

## 레코드 문서의 기본 템플릿 {#base-template-of-a-document-of-record}

기본 템플릿은 레코드 문서에 스타일 및 모양 정보를 제공합니다. 자동 생성된 기록 문서의 기본 모양을 사용자 정의할 수 있습니다. 예를 들어, 회사 로고를 머리글에 추가하고, 문서 바닥글에 저작권 정보를 추가하려고 합니다. 기본 템플릿의 마스터 페이지는 레코드 템플릿 문서의 마스터 페이지로 사용됩니다. 마스터 페이지에는 페이지 머리글, 페이지 바닥글 및 레코드 문서에 적용할 수 있는 페이지 번호와 같은 정보가 있을 수 있습니다. 기록 문서의 자동 생성을 위해 기본 템플릿을 사용하여 레코드 문서에 이러한 정보를 적용할 수 있습니다. 기본 템플릿을 사용하면 필드의 기본 속성을 변경할 수 있습니다.

기본 [템플릿을 디자인할 때 기본 템플릿 규칙을](#base-template-conventions) 따르십시오.

## 기본 템플릿 규칙 {#base-template-conventions}

기본 템플릿은 레코드 문서의 머리글, 바닥글, 스타일 및 모양을 정의하는 데 사용됩니다. 머리글과 바닥글에는 회사 로고 및 저작권 텍스트와 같은 정보가 포함될 수 있습니다. 기본 템플릿의 첫 번째 마스터 페이지는 머리글, 바닥글, 페이지 번호 또는 기록 문서의 모든 페이지에 표시되어야 하는 기타 정보가 포함된 레코드 문서의 마스터 페이지로 복사되고 사용됩니다. 기본 템플릿 규칙을 준수하지 않는 기본 템플릿을 사용하는 경우 기본 템플릿의 첫 번째 마스터 페이지는 여전히 레코드 템플릿 문서에서 사용됩니다. 기본 템플릿을 규칙에 따라 디자인하고 기록 문서의 자동 생성에 사용하는 것이 좋습니다.

**마스터 페이지 규칙**

* 기본 템플릿에서 루트 하위 폼의 이름을 `AF_METATEMPLATE` 로 지정하고 마스터 페이지의 이름을 로 지정해야 합니다 `AF_MASTERPAGE`.

* 루트 하위 폼 `AF_MASTERPAGE` 아래에 있는 이름이 있는 마스터 페이지에는 머리글, 바닥글 및 스타일 정보를 추출하기 위한 기본 설정이 `AF_METATEMPLATE` 있습니다.

* 없는 `AF_MASTERPAGE` 경우 기본 템플릿에 있는 첫 번째 마스터 페이지가 사용됩니다.

**필드에 대한 스타일 지정 규칙**

* 레코드 문서의 필드에 스타일을 적용하기 위해 기본 템플릿은 루트 하위 폼 아래의 `AF_FIELDSSUBFORM` 하위 폼에 있는 필드를 제공합니다 `AF_METATEMPLATE` .

* 이러한 필드의 속성은 레코드 문서의 필드에 적용됩니다. 이러한 필드는 `AF_<name of field in all caps>_XFO` 이름 지정 규칙을 따라야 합니다. 예를 들어 확인란 필드 이름은 `AF_CHECKBOX_XFO`다음과 같아야 합니다.

기본 템플릿을 만들려면 AEM Designer에서 다음을 수행합니다.

1. 파일 **> 새로 만들기를**&#x200B;클릭합니다.
1. 템플릿 **기준** 옵션을 선택합니다.

1. 양식 - **기록 문서 범주를** 선택합니다.
1. DoR **기본 템플릿을 선택합니다**.
1. 다음을 **클릭하고** 필요한 정보를 제공합니다.

1. (선택 사항) 기록 문서의 필드에 적용할 필드의 스타일 및 모양을 수정합니다.
1. 양식을 저장합니다.

이제 저장된 양식을 기록 문서의 기본 템플릿으로 사용할 수 있습니다.
기본 템플릿에 있는 스크립트를 수정하거나 제거하지 마십시오.

**기본 템플릿 수정**

* 기본 템플릿의 필드에 스타일을 적용하지 않는 경우 기본 템플릿에 대한 업그레이드가 자동으로 선택되도록 기본 템플릿에서 해당 필드를 제거하는 것이 좋습니다.
* 기본 템플릿을 수정하는 동안 스크립트를 제거, 추가 또는 수정하지 마십시오.

>[!NOTE]
>
>규칙을 사용하여 그리고 위의 단계를 엄격히 따르는 기본 템플릿을 디자인합니다.

## 기록 문서 템플릿 구성 {#document-of-record-template-configuration}

양식의 레코드 템플릿 문서를 구성하여 고객이 인쇄된 양식의 사본을 다운로드할 수 있도록 합니다. XDP 파일은 레코드 템플릿 문서로 사용됩니다. 고객 다운로드 기록 문서는 XDP 파일에 지정된 레이아웃에 따라 형식이 지정됩니다.

적응형 양식에 대한 기록 문서를 구성하려면 다음 단계를 수행하십시오.

1. AEM 작성자 인스턴스에서 양식 > 양식 **및 문서를 클릭합니다.**
1. 양식을 선택하고 속성 보기를 **클릭합니다**.
1. 속성 창에서 양식 모델을 **누릅니다**.
양식을 만들 때 양식 모델을 선택할 수도 있습니다.

   >[!NOTE]
   >
   >양식 모델 탭에서 스키마 **또는** 없음을 **선택** 드롭다운에서 **** 선택해야합니다. **[!UICONTROL 양식 템플릿을 양식 모델로 사용하는 XFA 기반 양식 또는 적응형 양식에는 기록 문서가 지원되지 않습니다.]**

1. 양식 모델 탭의 레코드 템플릿 구성 문서 섹션에서 다음 옵션 중 하나를 선택합니다.

   **없음** 양식에 대한 기록 문서를 구성하지 않으려면 이 옵션을 선택합니다.

## 기록 문서에서 브랜딩 정보 사용자 정의 {#customize-the-branding-information-in-document-of-record}

기록 문서를 생성하는 동안 기록 문서 탭에서 기록 문서에 대한 브랜드 정보를 변경할 수 있습니다. 기록 문서 탭에는 로고, 모양, 레이아웃, 머리글 및 바닥글, 부인 및 선택되지 않은 확인란 및 라디오 단추 옵션을 포함할지 여부와 같은 옵션이 포함되어 있습니다.

기록 문서 탭에서 입력한 브랜드 정보를 현지화하려면 브라우저의 로케일이 적절하게 설정되도록 해야 합니다. 기록 문서의 브랜딩 정보를 사용자 정의하려면 다음 단계를 완료하십시오.

1. 기록 문서에서 패널(루트 패널)을 선택한 다음 ![구성을](assets/configure.png)누릅니다.
1. Dortab을 ![누릅니다](assets/dortab.png). 기록 문서 탭이 나타납니다.
1. 레코드 문서를 렌더링하기 위한 기본 템플릿이나 사용자 지정 템플릿을 선택합니다. 기본 템플릿을 선택하면 템플릿 드롭다운 아래에 레코드 문서의 축소판 미리 보기가 나타납니다.

   ![브랜딩 템플릿](assets/brandingtemplate.png)

   사용자 지정 템플릿을 선택하려는 경우 AEM Forms 서버에서 XDP를 찾아봅니다. AEM Forms 서버에 없는 템플릿을 사용하려면 먼저 XDP를 AEM Forms 서버에 업로드해야 합니다.

1. 기본 템플릿을 선택하는지 사용자 지정 템플릿을 선택하는지에 따라 다음 속성 중 일부 또는 전체가 레코딩 문서 탭에 표시됩니다. 다음 사항을 적절히 지정합니다.

   * **로고 이미지**:적응형 양식의 로고 이미지를 사용하거나 DAM에서 로고 이미지를 선택하거나 컴퓨터에서 로고 이미지를 업로드할 수 있습니다.
   * **양식 제목**
   * **머리글 텍스트**
   * **면책 조항 레이블**
   * **면책조항**
   * **면책조항 텍스트**
   * **강조 색상**:문서 또는 PDF 레코딩에서 머리글 텍스트와 구분선이 렌더링되는 색상
   * **글꼴 모음**:기록 PDF 문서의 글꼴 모음
   * **확인란 및 라디오 단추 구성 요소의 경우 선택한 값만 표시합니다.**
   * **다중 선택 값에 대한 구분 기호**
   * **데이터 모델에 바인딩되지 않은 양식 개체 포함**
   * **기록 문서에서 숨김 필드 제외**
   * **패널 설명 숨기기**
   >[!NOTE]
   >
   >6.3 이전 버전의 디자이너를 사용하여 만든 응용 양식 템플릿을 사용하는 경우 강조 색상 및 글꼴 모음 속성이 작동하려면 루트 하위 폼 아래에 응용 양식 템플릿에 다음 내용이 있는지 확인합니다.

   ```xml
   <proto>
   <font typeface="Arial"/>
   <fill>
   <color value="4,166,203"/>
   </fill>
   <edge>
   <color value="4,166,203"/>
   </edge>
   </proto>
   ```

1. 브랜딩 변경 내용을 저장하려면 완료를 누릅니다.

## 기록 문서 설정 {#document-of-record-settings}

기록 문서 설정을 사용하면 기록 문서에 포함할 옵션을 선택할 수 있습니다. 예를 들어 은행은 양식에 이름, 연령, 주민등록번호 및 전화 번호를 허용합니다. 이 양식에서는 은행 계좌 번호와 지점 상세내역을 생성합니다. 기록 문서에 이름, 주민등록번호, 은행 계좌 및 지점 상세내역만 표시하도록 선택할 수 있습니다.

구성 요소의 기록 설정 문서는 해당 속성 아래에서 사용할 수 있습니다. 구성 요소의 속성에 액세스하려면 구성 요소를 선택하고 오버레이에서 ![cmppr](assets/cmppr.png) 을 클릭합니다. 속성은 세로 막대에 나열되며, 이 목록에서 다음 설정을 찾을 수 있습니다.

**필드 수준 설정**

* **기록 문서에서 제외**:속성을 true로 설정하면 레코드 문서에서 필드가 제외됩니다. 이 속성은 스크립트 가능 속성으로서 이름이 `excludeFromDoR`지정됩니다. 숨김 **** 양식 수준 속성의 경우 DoR에서 필드 제외에 따라 해당 동작이 달라집니다.

* **** 패널로 표시:패널에 필드가 6개 미만인 경우 속성을 설정하면 패널이 레코드 문서에 표로 표시됩니다. 패널에만 적용 가능합니다.
* **** 기록 문서에서 제목 제외:속성을 설정하면 패널/테이블의 제목이 기록 문서에서 제외됩니다. 패널 및 테이블에만 해당됩니다.
* **** 기록 문서에서 설명 제외:속성을 설정하면 패널/테이블의 설명이 레코드 문서에서 제외됩니다. 패널 및 테이블에만 해당됩니다.

**양식 수준 설정**

* **** DoR에 언바운드 필드 포함:속성 설정에는 기록 문서의 스키마 기반 적응형 양식의 바인딩되지 않은 필드가 포함됩니다. 기본적으로 true입니다.
* **** 숨겨진 경우 DoR에서 필드 제외:속성이 true가 아닌 경우 속성을 설정하면 &quot;기록 문서에서 제외&quot; 필드 수준 속성의 동작이 무시됩니다. 양식 제출 시 필드가 숨겨진 경우, &quot;기록 문서에서 제외&quot; 속성이 설정되지 않은 경우 속성이 true로 설정된 경우 레코드 문서에서 제외됩니다.

## 기록 문서 작업 시 주요 고려 사항 {#key-considerations-when-working-with-document-of-record}

적응형 양식의 기록 문서를 작업할 때 다음과 같은 고려 사항과 제한 사항을 염두에 두십시오.

* 기록 템플릿 문서는 리치 텍스트를 지원하지 않습니다. 따라서 정적 적응형 양식이나 최종 사용자가 입력한 정보에 포함된 모든 리치 텍스트는 기록 문서에 일반 텍스트로 표시됩니다.
* 적응형 양식의 문서 조각은 기록 문서에 나타나지 않습니다. 그러나 적응형 양식 조각이 지원됩니다.
* 기록 문서는 인쇄 용도로만 사용됩니다.
* XML 스키마 기반 적응형 양식에 대해 생성된 레코드 문서의 콘텐츠 바인딩은 지원되지 않습니다.
* XML 스키마 기반 적응형 양식에 대해 생성된 레코드 문서의 콘텐츠 바인딩은 지원되지 않습니다.
* 지역화된 버전의 기록 문서는 사용자가 기록 문서의 렌더링을 요청할 때 로케일에 대한 요청 시 생성됩니다. 기록 문서의 현지화는 적응형 양식의 현지화와 함께 이루어집니다. 레코드 및 적응형 양식의 문서 현지화에 대한 자세한 내용은 AEM [번역 워크플로우를 사용하여 적응형 양식 및 기록](/help/forms/using/using-aem-translation-workflow-to-localize-adaptive-forms.md)문서를 현지화하기를 참조하십시오.
