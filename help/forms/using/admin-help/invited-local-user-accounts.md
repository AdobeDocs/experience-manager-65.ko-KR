# 초대된 사용자 계정 및 로컬 사용자 계정 관리 {#managing-invited-and-local-user-accounts}

>[!NOTE]
> 
> 사용자에게 관리자 콘솔에 액세스할 수 있는 관리자 권한이 있는지 확인하십시오.

초대된 사용자 및 로컬 사용자 페이지에서 초대된 사용자 및 로컬 사용자를 관리합니다. 이 페이지는 다음과 같은 요구 사항을 충족하는 경우에만 표시됩니다.

* 문서 보안 초대된 사용자 및 로컬 사용자 관리 역할과 관리 콘솔 사용자 역할이 할당된 관리자입니다. ([역할 만들기 및 구성](/help/forms/using/admin-help/creating-configuring-roles.md#creating-and-configuring-roles)을 참조하십시오.)
* 초대된 사용자 등록이 활성화되어 있습니다. ([초대된 사용자 등록 구성](/help/forms/using/admin-help/configuring-client-server-options.md#configuring-invited-user-registration)을 참조하십시오.)

초대된 사용자 및 로컬 사용자 페이지에는 초대된 사용자 및 로컬 사용자 계정을 검색하고, 보고, 편집하고, 잠그거나 잠금 해제하고, 삭제하는 데 사용할 수 있는 두 개의 탭이 있습니다.

초대된 사용자에게 등록 이메일을 수동으로 보낼 수도 있습니다. 예를 들어 이메일에서 승인한 등록 기간이 종료되어 사용자가 URL을 사용하여 등록할 수 없는 경우 이 작업을 수행하는 것이 좋습니다. 이 경우 초대된 사용자에게 등록 이메일을 다시 보낼 수 있습니다. 초대된 사용자가 계정을 등록하고 활성화하면 해당 사용자는 로컬 사용자가 됩니다.

>[!NOTE]
>
>문서 보안에서 참조하는 LDAP 디렉터리를 통해 초대된 사용자를 직접 추가할 수도 있고 사용자 또는 관리자가 정책을 만들거나 편집할 때 새 사용자를 초대하면 등록 초대 이메일이 시작됩니다. 초대된 사용자 등록 페이지에서 초대된 사용자 등록 활성화 옵션을 활성화하면 사용자가 새로 초대된 사용자를 정책에 추가할 수 있습니다.

## 초대된 사용자 추가 {#add-an-invited-user}

초대된 사용자 계정을 한 번에 하나 이상 문서 보안에 추가할 수 있습니다. 초대된 사용자 계정을 추가하려면 사용자 이메일 주소가 필요합니다. 사용자를 추가하면 문서 보안에서 사용자에게 등록을 요청하는 등록 이메일을 보냅니다.

1. 관리 콘솔에서 서비스 > 문서 보안 > 초대된 사용자 및 로컬 사용자를 클릭한 후 새 사용자 초대를 클릭합니다.
1. 초대할 사용자의 이메일 주소를 입력합니다. 여러 주소를 쉼표로 구분하여 한 줄에 입력하십시오.

   초대된 사용자 등록을 활성화할 때 만든 메시지가 사용자에게 전송됩니다. ([초대된 사용자 등록 구성](/help/forms/using/admin-help/configuring-client-server-options.md#configuring-invited-user-registration)을 참조하십시오.)

1. 확인을 클릭합니다.

## 로컬 사용자 정보 보기 {#view-information-about-a-local-user}

이름, 이메일 주소, 조직, 등록 상태, 도메인을 포함한 로컬 사용자 정보를 볼 수 있습니다.

1. 관리 콘솔에서 서비스 > 문서 보안 > 초대된 사용자 및 로컬 사용자를 클릭한 후 새 사용자 초대를 클릭합니다.
1. 로컬 사용자 탭을 클릭하고 로컬 사용자 관리 페이지에서 확인할 사용자의 이메일 주소를 클릭합니다.

   사용자 세부 정보가 표시되며 사용자 암호를 재설정하고 계정을 비활성화할 수 있습니다.

## 등록되지 않은 외부 사용자에게 이메일 보내기 {#send-an-email-to-an-unregistered-external-user}

초대된 사용자를 추가하면 문서 보안에서 해당 사용자에게 등록 이메일 요청을 자동으로 보냅니다. 아직 등록하지 않은 초대된 사용자에게 보낼 등록 이메일을 수동으로 생성할 수도 있습니다. 예를 들어 초대된 사용자의 등록 이메일이 만료되는 경우 새 초대를 보내려면 이 작업을 수행하는 것이 좋습니다.

1. 관리 콘솔에서 서비스 > 문서 보안 > 초대된 사용자 및 로컬 사용자를 클릭합니다.
1. 사용자 목록에서 등록 이메일을 보낼 각 사용자의 확인란을 선택한 후 초대 이메일 다시 보내기를 클릭합니다.
1. 선택한 사용자 목록을 검토하고 확인을 클릭합니다.

## 로컬 사용자 암호 재설정 {#reset-a-local-user-password}

문서 보안에 등록한, 활성화된 초대된 사용자가 암호를 잊어버린 경우 암호를 재설정할 수 있습니다. 암호를 재설정하면 사용자의 새로운 임시 암호가 포함된 이메일이 생성됩니다.

초대된 사용자 등록 프로세스를 활성화하면 사용자에게 암호를 재설정하라는 이메일 메시지가 전송됩니다. ([초대된 사용자 등록 구성](/help/forms/using/admin-help/configuring-client-server-options.md#configuring-invited-user-registration)을 참조하십시오.)

1. 관리 콘솔에서 서비스 > 문서 보안 > 초대된 사용자 및 로컬 사용자를 클릭하고 로컬 사용자 탭을 클릭합니다.
1. 사용자 목록에서 해당 사용자를 선택합니다.
1. 로컬 사용자 관리 페이지에서 암호 재설정을 클릭하고 확인을 클릭합니다. 새 암호가 포함된 암호 재설정 이메일이 사용자에게 전송됩니다.

## 사용자 계정 활성화 또는 비활성화 {#enable-or-disable-a-user-account}

로컬 사용자 계정을 비활성화하여 사용자가 문서 보안에 로그인하는 것을 일시적으로 제한할 수 있습니다. 계정을 비활성화하면 사용자는 정책으로 보호된 문서를 사용하거나, 정책을 만들거나, 정책을 적용할 수 없습니다.

현재 비활성화된 로컬 사용자 계정을 활성화할 수 있습니다. 등록됨으로 나열된 초대된 사용자 계정은 활성화할 수 없습니다. 등록 상태는 초대된 사용자가 등록되었지만, 아직 활성화 이메일에 포함된 링크를 사용하여 계정을 활성화하지 않았음을 나타냅니다.

**사용자 계정 제한**

1. 관리 콘솔에서 서비스 > 문서 보안 > 초대된 사용자 및 로컬 사용자를 클릭하고 로컬 사용자 탭을 클릭합니다.
1. 사용자 목록에서 해당 사용자를 선택합니다.
1. 로컬 사용자 세부 정보 페이지에서 계정 비활성화를 클릭합니다.

**사용자 계정 복구**

1. 초대된 사용자 및 로컬 사용자를 클릭하고 로컬 사용자 탭을 클릭합니다.
1. 사용자 목록에서 해당 사용자를 선택합니다.
1. 로컬 사용자 세부 정보 페이지에서 계정 활성화를 클릭합니다.

## 초대된 사용자 계정 제거 {#remove-an-invited-user-account}

문서 보안에서 초대된 사용자 계정을 삭제할 수 있습니다. 예를 들어 사용자가 개인 이메일 계정 정보를 변경하는 경우 계정을 삭제하는 것이 좋습니다.

사용자 계정을 삭제하는 경우 관리자 또는 다른 관리자만 초대된 사용자 페이지에서 초대된 사용자 추가 옵션을 선택하여 해당 계정을 복구할 수 있습니다. 사용자는 삭제된 사용자 계정을 정책에 추가할 수 없으며 해당 방법을 통해 초대 프로세스를 시작할 수 없습니다.

>[!NOTE]
>
>AEM Forms 사용자 관리 인터페이스를 통해 삭제된 초대된 사용자는 다음 절차를 따라 다시 삭제하기 전까지는 다시 초대할 수 없습니다.

1. 관리 콘솔에서 서비스 > 문서 보안 > 초대된 사용자 및 로컬 사용자를 클릭하고 초대된 사용자 탭을 클릭합니다.
1. 한 명 이상의 사용자 옆에 있는 확인란을 선택하고 삭제를 클릭한 후 확인을 클릭합니다.

## 초대된 사용자 계정 검색 {#search-for-an-invited-user-account}

이메일 주소를 사용하여 초대된 사용자 계정을 검색할 수 있습니다.

1. 관리 콘솔에서 서비스 > 문서 보안 > 초대된 사용자 및 로컬 사용자를 클릭합니다.
1. 이메일 찾기 상자에 사용자의 이메일 주소를 입력한 후 찾기를 클릭합니다.

## 로컬 사용자 계정 검색 {#search-for-a-local-user-account}

사용자의 이메일 주소 또는 이름과 도메인을 사용하여 로컬 사용자를 검색할 수 있습니다.

1. 관리 콘솔에서 서비스 > 문서 보안 > 초대된 사용자 및 로컬 사용자를 클릭하고 로컬 사용자 탭을 클릭합니다.
1. 찾기 상자에 검색 기준을 입력하고 이름 또는 이메일을 선택한 후 찾기를 클릭합니다.

## 로컬 사용자 계정 제거 {#remove-a-local-user-account}

문서 보안에서 로컬 사용자 계정을 삭제할 수 있습니다. 예를 들어 사용자가 개인 이메일 계정 정보를 변경하는 경우 계정을 삭제하는 것이 좋습니다.

1. 관리 콘솔에서 서비스 > 문서 보안 > 초대된 사용자 및 로컬 사용자를 클릭하고 로컬 사용자 탭을 클릭합니다.
1. 한 명 이상의 사용자 옆에 있는 확인란을 선택하고 삭제를 클릭한 후 확인을 클릭합니다.

## 사용자 목록 정렬 {#sort-the-user-list}

열 제목을 기준으로 사용자 목록을 정렬하면 사용자를 더 쉽게 찾을 수 있습니다. 열 제목 옆에 있는 삼각형 아이콘은 현재 어떤 열을 기준으로 정렬 중인지 알려줍니다.

* 위를 카리키는 삼각형은 오름차순을 나타냅니다.
* 아래를 가리키는 삼각형은 내림차순을 나타냅니다.

   1. 관리 콘솔에서 서비스 > 문서 보안 > 초대된 사용자 및 로컬 사용자를 클릭합니다.
   1. 초대된 사용자를 정렬하려면 초대된 사용자 탭을 클릭하고 해당 열 제목을 클릭합니다.
   1. 로컬 사용자를 정렬하려면 로컬 사용자 탭을 클릭하고 해당 열 제목을 클릭합니다.
