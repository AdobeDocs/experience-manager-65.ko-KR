# 작업 관리자 데이터베이스에서 레코드 제거 {#purge-records-from-the-job-manager-database}

>[!NOTE]
> 
> 사용자에게 관리자 콘솔에 액세스할 수 있는 관리자 권한이 있는지 확인하십시오.

장기 프로세스를 호출할 때 생성되는 프로세스 데이터는 너무 커질 수 있으며, 이로 인해 AEM Forms 성능이 저하되고 불필요한 디스크 공간이 사용됩니다. 더 이상 레코드가 필요하지 않으면 프로세스 데이터를 제거하는 것이 좋습니다.

관리 콘솔을 사용하여 더 이상 사용되지 않는 레코드를 한 번만 제거하거나 정기 자동 제거를 예약할 수 있습니다. 더 이상 사용되지 않는 레코드를 제거하는 다른 방법은 [프로세스 데이터 제거](/help/forms/using/admin-help/purging-process-data.md#purging-process-data)에 설명되어 있습니다.

**제거 작업 스케줄러 페이지 액세스**

1. 관리 콘솔에서 페이지 오른쪽 상단에 있는 상태 모니터를 클릭합니다.
1. 제거 작업 스케줄러 탭을 클릭합니다.

현재 예약된 제거에 대한 정보는 제거 작업 스케줄러 정보 상자에 표시됩니다.

>[!NOTE]
>
>스케줄러 중지를 클릭하면 향후 예약된 제거 작업이 중지되지만, 이미 진행 중인 제거 작업은 중지되지 않습니다.

**일회성 제거 예약**

1. 한 번만을 선택합니다.
1. 제거 완료된 레코드 필터 영역에서 레코드가 더 이상 사용되지 않아 제거할 준비가 된 것으로 간주되는 일수 또는 주수를 지정합니다.

   >[!NOTE]
   >
   >완료되지 않은 프로세스와 관련된 레코드는 지정된 기간보다 오래되었더라도 제거되지 않습니다.

1. 제거 작업을 수행할 시기를 지정합니다. 현재 일자 및 시간 사용 확인란을 선택하거나 확인란을 선택 취소하고 캘린더 및 시계 아이콘을 클릭하여 제거를 수행할 일자 및 시간을 지정합니다.

   >[!NOTE]
   >
   >과거 시작 일자 및 시간을 지정하면 스케줄러 시작을 클릭하는 즉시 제거가 수행됩니다.

1. 스케줄러 시작을 클릭합니다. 이전에 예약된 스케줄러 설정은 새 설정으로 대체됩니다.

**자동 제거 일정 구성**

1. 반복 주기를 선택하고 제거 간격의 일수 또는 주수를 지정합니다.
1. 제거 완료된 레코드 필터 영역에서 레코드가 더 이상 사용되지 않아 제거할 준비가 된 것으로 간주되는 일수 또는 주수를 지정합니다. 이 값을 `0`으로 설정할 수 없습니다.

   >[!NOTE]
   >
   >완료되지 않은 프로세스와 관련된 레코드는 지정된 기간보다 오래되었더라도 제거되지 않습니다.

1. 제거 작업을 시작할 시기를 지정합니다. 현재 일자 및 시간 사용 확인란을 선택하거나 확인란을 선택 취소하고 캘린더 및 시계 아이콘을 클릭하여 제거를 수행할 일자 및 시간을 지정합니다.

   >[!NOTE]
   >
   >과거 시작 일자 및 시간을 지정하면 AEM Forms에서는 지정한 일자를 기준으로 논리적인 다음 시작 일자를 계산합니다. 예를 들어 4월 7일부터 매주 제거 작업을 수행하도록 예약했는데 현재 일자가 4월 9일이면 첫 번째 제거 작업은 4월 14일에 수행됩니다.

1. 스케줄러 시작을 클릭합니다. 이전에 예약된 스케줄러 설정은 새 설정으로 대체됩니다.
