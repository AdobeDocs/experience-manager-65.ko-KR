# 그룹 만들기 및 구성{#creating-and-configuring-groups}

>[!NOTE]
> 
> 사용자에게 관리자 콘솔에 액세스할 수 있는 관리자 권한이 있는지 확인하십시오.

사용자 그룹을 만들면 개별 사용자가 아닌 그룹에 역할을 할당할 수 있습니다.

그룹은 두 가지 유형이 있습니다. 그룹을 수동으로 만들고 사용자와 다른 그룹을 해당 그룹에 추가할 수 있습니다. 지정된 규칙 집합을 충족하는 모든 사용자를 자동으로 포함하는 동적 그룹을 만들 수도 있습니다.

사용자가 여러 그룹에 속해 있거나(예: 500개 이상) 그룹이 깊이 중첩되어 있는 경우(예: 30개 수준) 응답 시간이 느려질 수 있습니다. 이러한 문제가 발생하는 경우 특정 도메인에서 정보를 미리 가져오도록 AEM Forms를 구성할 수 있습니다. ([도메인 정보를 미리 가져오도록 AEM Forms 구성](/help/forms/using/admin-help/configure-aem-forms-prefetch-domain.md#configure-aem-forms-to-prefetch-domain-information)을 참조하십시오.)

## 수동으로 그룹 만들기 {#create-a-group-manually}

그룹을 수동으로 만들면 사용자와 다른 그룹을 해당 그룹에 추가하고 그 그룹에 역할을 할당할 수 있습니다. 그룹을 상위 그룹과 연결할 수도 있습니다.

콘텐츠 서비스(더 이상 사용되지 않음)를 사용하는 경우 도메인 관리 페이지에서 등록된 외부 주체 스토리지 공급자로 사용자 및 그룹을 푸시하려면 이 옵션 선택 옵션을 선택하여 콘텐츠 서비스(더 이상 사용되지 않음)에서 만든 모든 새 사용자 또는 그룹에 대한 정보를 푸시할 수 있습니다.

1. 관리 콘솔에서 설정 > 사용자 관리 > 사용자 및 그룹을 클릭한 후 새 그룹을 클릭합니다.
1. 일반 설정 섹션을 완료하고 다음을 클릭합니다. 정식 이름과 그룹 이름은 필수 속성입니다.

   정식 이름은 그룹의 고유 식별자입니다. 도메인의 각 그룹 및 사용자는 고유한 정식 이름을 가져야 합니다. 시스템 생성 확인란을 선택하여 사용자 관리에서 고유한 값을 할당하도록 하거나 확인란을 선택 취소하고 정식 이름에 대한 사용자 정의 값을 지정합니다.

   정식 이름에는 `sample_group`과 같은 밑줄 문자(_)를 사용하지 마십시오. 정식 이름을 기준으로 그룹을 검색하는 경우 밑줄 문자가 포함된 그룹은 반환되지 않습니다.

1. 이 새 그룹에 사용자 및 그룹을 추가하려면 사용자/그룹 찾기를 클릭하고 다음 작업을 수행합니다.

   * 찾기 상자에 검색 기준을 입력합니다.
   * 검색 대상 목록에서 사용자, 그룹 또는 사용자 및 그룹을 선택합니다.
   * 사용 목록에서 이름, 이메일 또는 사용자 ID를 선택합니다.
   * 도메인을 선택하고 표시할 항목 수를 선택한 후 찾기를 클릭합니다.
   * 검색 결과에서 이 새 그룹에 추가할 사용자 및 그룹의 확인란을 선택하고 확인을 클릭합니다.

1. 다음을 클릭합니다.
1. 이 새 그룹을 다른 기존 그룹에 추가하려면 그룹 찾기를 클릭하고 다음 작업을 수행합니다.

   * 찾기 상자에 검색 기준을 입력합니다.
   * 도메인을 선택하고 표시할 항목 수를 선택한 후 찾기를 클릭합니다.
   * 검색 결과에서 새 그룹이 속한 그룹의 확인란을 선택하고 확인을 클릭합니다.

1. 다음을 클릭합니다.
1. 그룹에 역할을 할당하려면 역할 찾기를 클릭하고 그룹에 할당할 각 역할의 확인란을 선택한 후 확인을 클릭합니다. 그룹에 속한 사용자는 그룹 수준에서 할당된 역할을 상속받습니다.
1. 마침을 클릭합니다.

## 동적 그룹 만들기 {#create-a-dynamic-group}

동적 그룹에서는 그룹에 속한 사용자를 개별적으로 선택하지 않습니다. 대신 일련의 규칙을 지정하면 해당 규칙을 충족하는 모든 사용자가 자동으로 동적 그룹에 추가됩니다.

다음 두 가지 방법 중 하나를 사용하여 동적 그룹을 만들 수 있습니다.

* @adobe.com과 같은 이메일 도메인을 기반으로 동적 그룹을 자동으로 만들 수 있습니다. 이 기능을 활성화하면 사용자 관리에서 AEM Forms 데이터베이스에 있는 고유한 이메일 도메인마다 동적 그룹을 만듭니다. Cron 표현식을 사용하여 사용자 관리가 AEM Forms 데이터베이스에서 새 이메일 도메인을 검색하는 빈도를 지정합니다. 이러한 동적 그룹은 DefaultDom 로컬 도메인에 추가되고 &#39;*`[email domain]`* 메일 ID가 있는 모든 사용자&#39;라는 이름이 지정됩니다.
* 사용자의 이메일 도메인, 설명, 정식 이름, 도메인 이름을 포함한 지정된 기준에 따라 동적 그룹을 만듭니다. 동적 그룹에 속하려면 사용자가 지정된 모든 기준을 충족해야 합니다. &#39;OR&#39; 조건을 설정하려면 두 개의 개별 동적 그룹을 만들고 둘 다 로컬 그룹에 추가합니다. 예를 들어 이 접근 방식을 사용하여 @adobe.com 이메일 도메인에 속하거나 정식 이름에 ou=adobe.com이 포함된 사용자 그룹을 만듭니다. 하지만 사용자가 반드시 두 가지 조건을 모두 충족할 필요는 없습니다.

동적 그룹에는 사용자만 포함됩니다. 다른 그룹은 포함할 수 없습니다. 하지만 동적 그룹은 상위 그룹에 속할 수 있습니다.

### 이메일 도메인에 따라 자동으로 동적 그룹 만들기 {#automatically-create-dynamic-groups-based-on-email-domains}

1. 설정 > 사용자 관리 > 구성 > 고급 시스템 속성 구성을 클릭합니다.
1. 동적 그룹 자동 생성에서 확인란을 선택합니다.
1. 사용자 관리자가 새로운 이메일 도메인을 확인하는 시기를 지정합니다. 동적 그룹 생성은 도메인 동기화가 완료된 경우에만 논리적이므로 이 시간은 도메인 동기화 시간 이후여야 합니다.

   * 자동 동기화를 매일 활성화하려면 매일 발생 시간 상자에 24시간 형식으로 시간을 입력합니다. 설정을 저장하면 이 값이 Cron 표현식으로 변환되어 아래 상자에 표시됩니다.
   * 특정 요일이나 날짜 또는 특정 월에 동기화를 예약하려면 상자에 해당 Cron 표현식을 입력합니다. 기본값은 `0 00 4 ? * *`(매일 오전 4시에 확인)입니다.

     Cron 표현식 사용법은 Quartz 오픈 소스 작업 예약 시스템 버전 1.4.0을 기반으로 합니다.

1. 저장을 클릭합니다.

### 지정된 기준에 따라 동적 그룹 만들기 {#create-a-dynamic-group-based-on-specified-criteria}

1. 관리 콘솔에서 설정 > 사용자 관리 > 사용자 및 그룹을 클릭합니다.
1. 새로운 동적 그룹을 클릭합니다.
1. 일반 설정 섹션을 완료합니다. 그룹 이름은 필수 속성입니다. 구성된 모든 도메인에 그룹을 할당할 수 있습니다.
1. 동적 그룹 기준에서 동적 그룹을 채우는 데 사용되는 하나 이상의 속성을 지정합니다.

   >[!NOTE]
   >
   >이메일, 설명 및 정식 이름 속성은 Equals 연산자를 사용할 때 대소문자를 구분합니다. Starts With, Ends With, Contains 연산자를 사용할 때는 대소문자를 구분하지 않습니다.

   **이메일:** 사용자의 이메일 도메인입니다(예: `@adobe.com`).

   **설명:** 사용자 설명(예: &#39;컴퓨터 과학자&#39;)입니다.

   **정식 이름:** 사용자의 정식 이름입니다(예: `ou=adobe.com`).

   **도메인 이름:** 사용자가 속한 도메인 이름입니다(예: `DefaultDom`). 도메인 이름 속성은 Contains 연산자를 사용할 때 대소문자를 구분합니다. Starts With, Ends With, Equals 연산자를 사용할 때는 대소문자를 구분하지 않습니다.

1. 테스트를 클릭합니다. 테스트 페이지에 정의된 기준을 충족하는 처음 200명의 사용자가 표시됩니다. 닫기를 클릭합니다.
1. 테스트가 예상한 결과를 반환하면 다음을 클릭합니다. 그렇지 않으면 동적 그룹 기준을 편집하고 다시 테스트합니다.
1. 동적 그룹을 상위 그룹에 추가하려면 그룹 찾기를 클릭하고 다음 작업을 수행합니다.

   * 찾기 상자에 검색 기준을 입력합니다.
   * 도메인을 선택하고 표시할 항목 수를 선택한 후 찾기를 클릭합니다.
   * 검색 결과에서 동적 그룹이 속한 그룹의 확인란을 선택하고 확인을 클릭합니다.

1. 다음을 클릭합니다.
1. 동적 그룹에 역할을 할당하려면 역할 찾기를 클릭하고 그룹에 할당할 각 역할의 확인란을 선택한 후 확인을 클릭합니다. 그룹에 속한 사용자는 그룹 수준에서 할당된 역할을 상속받습니다.
1. 마침을 클릭합니다.

## 그룹에 대한 세부 정보 보기 {#view-details-about-a-group}

1. 관리 콘솔에서 설정 > 사용자 관리 > 사용자 및 그룹을 클릭합니다.
1. 검새 대상 목록에서 그룹을 선택한 후 찾기를 클릭합니다. 검색 결과는 페이지 하단에 나열됩니다. 열 제목을 클릭하면 목록을 정렬할 수 있습니다.
1. 그룹 이름을 클릭하여 해당 그룹의 세부 정보를 표시합니다. 그룹 세부 정보 페이지가 나타납니다.
1. 그룹의 직접 멤버를 보려면 하위 주체를 클릭합니다.

## 그룹 편집 {#edit-a-group}

1. 관리 콘솔에서 설정 > 사용자 관리 > 사용자 및 그룹을 클릭합니다.
1. 편집할 그룹을 찾으려면 다음 작업을 수행합니다.

   * 찾기 상자에 검색 기준을 입력합니다.
   * 사용 목록에서 이름 또는 이메일을 선택합니다.
   * 검색 대상 목록에서 그룹을 선택합니다.
   * 도메인을 선택하고 표시할 항목 수를 선택한 후 찾기를 클릭합니다.
   * 검색 결과에서 편집할 그룹 이름을 클릭합니다.

1. 세부 정보 탭에서 일반 설정을 편집하고 저장을 클릭합니다.
1. 연결된 그룹을 편집하려면 상위 그룹 탭을 클릭하고 다음 작업을 수행합니다.

   * 연결에 추가할 그룹을 찾으려면 그룹 찾기를 클릭하고 검색 정보를 입력합니다.
   * 그룹을 추가하려면 추가할 그룹의 확인란을 선택하고 확인을 클릭한 후 저장을 클릭합니다.
   * 연결된 그룹을 삭제하려면 삭제할 그룹의 확인란을 선택하고 삭제를 클릭한 후 확인을 클릭하고 저장을 클릭합니다.

1. 그룹에 속한 사용자 및 그룹을 편집하려면 하위 주체 탭을 클릭하고 다음 작업을 수행합니다.

   * 추가할 사용자 및 그룹을 찾으려면 사용자/그룹 찾기를 클릭하고 검색 정보를 입력합니다.
   * 사용자 또는 그룹을 추가하려면 해당 사용자 또는 그룹의 확인란을 선택하고 확인을 클릭한 후 저장을 클릭합니다.
   * 사용자 또는 그룹을 삭제하려면 해당 사용자 또는 그룹의 확인란을 선택하고 삭제를 클릭한 후 확인을 클릭하고 저장을 클릭합니다.

1. 역할 할당을 편집하려면 역할 할당 탭을 클릭하고 다음 작업을 수행합니다.

   * 그룹에 할당할 역할을 찾으려면 역할 찾기를 클릭합니다.
   * 역할을 추가하려면 역할의 확인란을 선택하고 확인을 클릭한 후 저장을 클릭합니다.
   * 역할을 할당 취소하려면 역할의 확인란을 선택하고 할당 취소를 클릭한 후 저장을 클릭합니다.

## 그룹 삭제 {#delete-a-group}

1. 관리 콘솔에서 설정 > 사용자 관리 > 사용자 및 그룹을 클릭합니다.
1. 찾기 목록에서 그룹을 선택한 후 찾기를 클릭합니다.
1. 삭제할 그룹의 확인란을 선택하고 삭제를 클릭한 후 확인을 클릭합니다.
