# 프로세스 관리 {#managing-processes}

프로세스 목록 페이지에는 사용자가 시작했거나 자동으로 시작된 프로세스가 표시됩니다.

1. 관리 콘솔에서 서비스 > Forms Workflow > Forms Workflow를 클릭합니다. 프로세스 목록에 다음 정보가 표시됩니다.

   **프로세스 이름 - 버전:** 워크벤치에 정의된 프로세스 이름입니다.

   **애플리케이션:** 워크벤치에 정의된 대로 프로세스가 속한 애플리케이션입니다.

   **상태:** 활성은 프로세스가 해당 프로세스 버전에 대해 활성화되었다는 것을 의미합니다. 비활성은 프로세스가 여전히 프로세스 인스턴스가 있는 이전 버전이라는 것을 의미합니다.

   **생성 일자:** 프로세스가 배포된 일자 및 시간입니다.

1. 프로세스 이름을 클릭하여 프로세스 인스턴스 페이지에서 해당 프로세스 인스턴스를 확인합니다.

## 프로세스 인스턴스 작업 {#working-with-process-instances}

프로세스 목록 페이지에서 프로세스 인스턴스 페이지에 액세스하면 선택한 프로세스의 모든 프로세스 인스턴스가 나열됩니다. 검색을 수행한 후 프로세스 인스턴스 페이지에 액세스하면 발견된 프로세스 인스턴스만 나열됩니다.

목록에는 각 프로세스 인스턴스에 대한 다음 정보가 표시됩니다.

**프로세스 ID:** 프로세스가 인스턴스화될 때(즉, 사용자 또는 자동화된 단계에 따라 프로세스가 시작될 때) Forms Workflow에서 할당하는 식별자입니다. 이 식별자를 사용하면 프로세스 인스턴스를 수명 주기 전체에서 추적할 수 있습니다.

**프로세스 이름 - 버전:** 워크벤치에 정의된 프로세스 이름입니다.

**상태:** 프로세스 인스턴스가 정상적으로 실행 중인지, 상태를 변경하는 중인지, 중지되었는지 여부를 나타냅니다. (프로세스 인스턴스 상태 정보를 참조하십시오.)

**생성 일자:** 프로세스 인스턴스가 생성된 일자 및 시간입니다.

**업데이트 일자:** 프로세스 인스턴스 상태가 마지막으로 변경된 일자 및 시간입니다.

프로세스 인스턴스 페이지에서 다음 작업을 수행할 수 있습니다.

* 프로세스 인스턴스를 선택하여 해당 작업 및 하위 프로세스와 같은 세부 정보를 봅니다. 프로세스 인스턴스를 선택하면 프로세스 인스턴스 세부 정보 페이지가 나타납니다.
* 프로세스 인스턴스를 일시 중단, 일시 중단 취소 또는 종료합니다.
* 프로세스 인스턴스를 검색합니다. 검색을 시작하려면 검색을 클릭하십시오.

### 프로세스 인스턴스 상태 정보 {#about-process-instance-statuses}

하위 프로세스를 포함한 프로세스 인스턴스의 상태는 다음과 같을 수 있습니다.

**완료:** 프로세스 인스턴스의 모든 분기와 작업이 완료되었습니다. 완료는 프로세스 인스턴스의 최종 상태입니다.

**완료 중:** 프로세스 인스턴스 상태가 완료로 변경되려고 합니다.

**시작됨:** 프로세스 인스턴스가 생성되었지만, 아직 실행되고 있지 않습니다. 시작됨은 프로세스 인스턴스의 첫 번째 상태입니다.

**실행 중:** 프로세스 인스턴스가 정상적으로 실행 중입니다. 자동화된 단계가 진행 중일 수도 있고 프로세스 인스턴스가 사용자 입력을 받고 있거나 사용자 상호 작용을 기다리고 있을 수도 있습니다.

**일시 중단됨:** 프로세스 인스턴스가 관리자 또는 프로세스 단계에 의해 일시 중단되었습니다. 상태가 변경될 때까지 추가 작업이 발생하지 않습니다.

**일시 중단 중:** 상태가 일시 중단됨으로 변경되려고 합니다. 작업이 일시 중단 요청을 무시하도록 설계되었지만 아직 완료되지 않은 경우 프로세스 인스턴스가 일시 중단되기 전에 해당 작업이 완료되어야 합니다.

**종료됨:** 프로세스 인스턴스가 관리자에 의해 종료되었습니다.

**종료 중:** 상태가 종료됨으로 변경되려고 합니다. 작업이 종료 요청을 무시하도록 설계되었지만 아직 완료되지 않은 경우 프로세스 인스턴스가 종료되기 전에 해당 작업이 완료되어야 합니다.

**일시 중단 취소 중:** 일시 중단된 후 상태가 실행 중으로 변경되려고 합니다.

>[!NOTE]
>
>프로세스 인스턴스 상태를 변경하라는 요청(예: 일시 중단 또는 종료)이 이루어지면 해당 요청은 Forms Workflow의 명령 대기열에 들어갑니다. 대기열 크기와 전반적인 처리 속도에 따라 페이지를 한 번 이상 다시 로드하기 전까지는 표시된 상태가 변경되지 않을 수 있습니다.

### 프로세스 인스턴스 일시 중단 또는 일시 중단 취소 {#suspend-or-unsuspend-process-instances}

문제를 해결해야 하는 경우 또는 외부 조건으로 인해 이후 단계에서 프로세스 인스턴스에 문제가 발생할 것으로 예상되는 경우 프로세스 인스턴스를 일시적으로 중단할 수 있습니다.

실행 중 상태의 프로세스 인스턴스를 일시 중단할 수 있습니다.

프로세스 인스턴스를 일시 중단하면 상태가 일시 중단 중으로 변경되었다가 일시 중단됨으로 변경되고 프로세스가 현재 작업에서 일시 중지됩니다. 프로세스 인스턴스는 상태가 일시 중단 취소됨으로 변경될 때까지 해당 상태를 유지합니다.

일시 중단됨 상태의 프로세스 인스턴스만 일시 중단 취소됨으로 변경할 수 있습니다.

프로세스 인스턴스의 일시 중단을 취소하면 상태가 실행 중으로 변경되고 일시 중단되었던 작업이 계속 진행됩니다.

다른 프로세스(하위 프로세스)의 호출 작업을 사용하여 해당 프로세스를 호출한 프로세스 인스턴스를 일시 중단하면 해당 하위 프로세스도 일시 중단됩니다.

1. 관리 콘솔에서 서비스 > Forms Workflow > Forms Workflow를 클릭합니다.
1. 프로세스 인스턴스 페이지에서 프로세스를 선택하고 일시 중단 또는 일시 중단 취소를 클릭합니다.

### 프로세스 인스턴스 종료 {#terminate-a-process-instances}

프로세스 인스턴스 작업이 중단되거나 다른 오류 조건이 발생한 경우 또는 프로세스 인스턴스의 실행을 강제로 중지해야 하는 경우 프로세스 인스턴스를 종료할 수 있습니다.

모든 상태의 프로세스 인스턴스를 종료할 수 있습니다.

프로세스 인스턴스를 종료하면 상태가 종료 중으로 변경되었다가 종료됨으로 변경되고 프로세스가 현재 작업에서 중지됩니다. 더 이상 작업이 실행되지 않으며 연결된 모든 작업이 종료됩니다.

1. 관리 콘솔에서 서비스 > Forms Workflow > Forms Workflow를 클릭합니다.
1. 프로세스 인스턴스 페이지에서 프로세스를 선택하고 종료를 클릭합니다.

## 프로세스 인스턴스 세부 정보 작업 {#working-with-process-instance-details}

프로세스 인스턴스 세부 정보 페이지는 프로세스 인스턴스 기록이 표시됩니다.

요약 영역에는 프로세스 인스턴스에 대한 기본 정보가 표시됩니다.

작업 탭에서는 프로세스 인스턴스의 각 작업이 다음 정보와 함께 완료된 순서대로 처음부터 끝까지 표시됩니다.

**작업 이름:** 워크벤치에 정의된 작업 이름입니다.

**상태:** 작업이 정상적으로 실행 중인지 또는 중지되었는지 여부를 나타냅니다. (프로세스 인스턴스 상태 정보를 참조하십시오.)

**분기 이름:** 워크벤치에 정의된 분기 이름입니다.

**시작 일자:** 작업이 시작된 일자 및 시간입니다.

**완료 일자:** 작업이 완료된 일자 및 시간입니다.

하위 프로세스는 다른 프로세스에 의해 시작되고 다른 프로세스와 독립적으로 실행되는 프로세스 인스턴스입니다. 하위 프로세스는 워크벤치에서 프로세스의 일부로 설계된 경우에만 표시됩니다. 하위 프로세스 탭에서는 각 하위 프로세스가 다음 정보와 함께 표시됩니다.

**프로세스 ID:** 프로세스가 인스턴스화될 때(즉, 사용자 또는 자동화된 단계에 따라 프로세스가 시작될 때) Forms Workflow에서 할당하는 양의 정수입니다. 이 식별자를 사용하면 프로세스 인스턴스를 수명 주기 전체에서 추적할 수 있습니다.

**프로세스 이름 - 버전:** Designer에 정의된 프로세스 이름입니다.

**상태:** 프로세스 인스턴스가 정상적으로 실행 중인지, 상태를 변경하는 중인지, 중지되었는지 여부를 나타냅니다. (프로세스 인스턴스 상태 정보를 참조하십시오.)

**생성 일자:** 하위 프로세스가 생성된 일자 및 시간입니다.

**업데이트 일자:** 하위 프로세스의 상태가 마지막으로 변경된 일자 및 시간입니다.

프로세스 인스턴스 세부 정보 페이지에서 다음 작업을 수행할 수 있습니다.

* 작업을 선택하여 세부 정보를 봅니다. 작업을 선택하면 작업 세부 정보 페이지가 나타납니다.
* 하위 프로세스를 선택하여 세부 정보를 봅니다. 하위 프로세스를 선택하면 프로세스 인스턴스 세부 정보 페이지가 나타납니다.
* 상태에 따라 작업이나 하위 프로세스를 종료하거나 재시도합니다.

### 작업 상태 정보 {#about-operation-statuses}

작업(프로세스 단계) 상태는 다음과 같을 수 있습니다.

**완료:** 작업이 완료되었습니다.

**실행 중:** 작업이 정상적으로 실행 중입니다. 사용자 입력을 받고 있거나, 사용자 상호 작용을 기다리고 있거나, 자동화된 단계가 진행 중일 수 있습니다.

**중단됨:** 작업을 처리하는 동안 문제가 발생했습니다. 중단된 작업 페이지에서 오류 또는 예외를 확인합니다.

**종료됨:** 작업이 관리자에 의해 종료되었습니다.

### 작업 또는 하위 프로세스 종료 {#terminate-operations-or-subprocesses}

작업이나 하위 프로세스가 중단되거나 다른 오류 조건이 발생한 경우 또는 작업이나 하위 프로세스의 실행을 강제로 중지해야 하는 경우 해당 작업이나 하위 프로세스를 종료할 수 있습니다.

실행 중 상태의 작업을 종료할 수 있습니다.

작업을 종료하면 상태가 종료됨으로 변경됩니다. 작업이 완료되지 않고 프로세스 인스턴스의 실행이 중지됩니다.

모든 상태의 하위 프로세스를 종료할 수 있습니다.

하위 프로세스를 종료하면 상태가 종료 중으로 변경되었다가 종료됨으로 변경되고 프로세스 인스턴스가 현재 작업에서 중지됩니다. 상위 프로세스 인스턴스는 계속 실행되지만, 하위 프로세스에서는 더 이상 작업이 실행되지 않습니다.

프로세스 다이어그램에 게이트웨이 요소가 있는 프로세스는 종료할 수 없습니다. 이러한 유형의 프로세스를 종료하려고 시도해도 게이트웨이 요소 내에 있는 작업에는 영향을 미치지 않습니다. 게이트웨이 요소 내에 있는 작업을 종료하려면 해당 작업을 직접 종료해야 합니다.

1. 프로세스 인스턴스 세부 정보 페이지에서 작업 탭이나 하위 프로세스 탭을 클릭합니다.
1. 작업이나 하위 프로세스를 선택하고 종료를 클릭합니다.

### 작업 재시도 {#retry-an-operation}

중단됨 상태의 작업을 재시도할 수 있습니다.

작업을 재시도하면 Forms Workflow에 작업을 다시 시작하라는 요청이 전송됩니다. 요청이 성공하면 상태가 실행 중으로 변경됩니다. 작업을 다시 시작할 수 없는 경우에는 중단됨 상태로 유지되며 해당 작업을 종료해야 할 수도 있습니다.

1. 프로세스 인스턴스 세부 정보 페이지에서 작업 탭을 클릭합니다.
1. 작업을 선택하고 재시도를 클릭합니다.

## 작업 처리 {#working-with-operations}

>[!NOTE]
> 
> 사용자에게 관리자 콘솔에 액세스할 수 있는 관리자 권한이 있는지 확인하십시오.

작업 세부 정보 페이지에는 프로세스에 포함된 단일 작업에 대한 요약과 현재 사용자 할당이 표시됩니다.

1. 관리 콘솔에서 서비스 > Forms Workflow > Forms Workflow를 클릭합니다.
1. 프로세스 이름을 클릭하여 해당 프로세스 인스턴스를 표시합니다. 프로세스 인스턴스를 클릭하여 프로세스 인스턴스 세부 정보 페이지를 표시한 후 작업을 선택하여 작업 세부 정보 페이지를 표시합니다.

   목록에는 다음과 같은 각 작업의 정보가 표시됩니다.

   **프로세스 이름 - 버전:** 워크벤치에 정의된 프로세스 이름입니다.

   **애플리케이션:** 워크벤치에 정의된 대로 프로세스가 속한 애플리케이션입니다.

   **상태:** 활성은 프로세스가 해당 프로세스 버전에 대해 활성화되었다는 것을 의미합니다. 비활성은 프로세스가 여전히 프로세스 인스턴스가 있는 이전 버전이라는 것을 의미합니다.

   **생성 일자:** 프로세스가 배포된 일자 및 시간입니다.
